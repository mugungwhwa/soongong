# Yieldo Terminal 모창 심사평 → SparkClaw 재활용 자산

> **이 문서는 PROJECT.md §1.2(창업자 자산) + SparkClaw 포지셔닝 부록이다.** Yieldo Terminal(모창 탈락 건)의 심사평에서 순공대장에 그대로 쓸 수 있는 신뢰 증거 / 반면교사 / 선제 방어 포인트를 추출한 것. 결론: 이 탈락은 위로거리가 아니라 **"순공대장 GTM이 더 깨끗한 베팅"이라는 외부 신호**다.

**작성일**: 2026-06-09
**출처**: 모창 Yieldo Terminal 심사평 (제3자 평가)

---

## 1. 한 줄 요약

Yieldo를 죽인 결함은 전부 **B2B 엔터프라이즈 SaaS의 고질병**(고객사별 맞춤 연동, 느린 신뢰 형성, 보안 게이트, 대형사 내재화 위협)이고, 순공대장은 그 함정을 **하나도 안 밟는다.** 같은 창업자, 더 깨끗한 베팅.

---

## 2. 재활용 자산 A — 창업자 신뢰성 제3자 인증 (PROJECT.md §1.2에 인용)

심사위원이 **공식적으로 인정한** 것 (베타 데이터 없이도 SparkClaw application gate에 쓸 수 있는 builder credibility 증거):

- 16년 게임 산업 마케팅·데이터 전략 경험이 **서비스 설계 전반에 반영**되어 있고, 도메인 이해도가 높다.
- 채널별 CAC·ROAS 단기 효율을 넘어 운영·실험·재무·시장을 **하나의 판단 체계로 통합**하려는 접근 = 베이지안 / A/B 사고의 외부 검증.
- "실무 중심 문제의식에서 출발"했다 = 문제 정의의 현실성 인정.

→ **SparkClaw application gate가 요구하는 "데이터 사고 능력 + 빌더 신뢰성"을 제3자가 문서로 확인해준 셈.** → §1.2 표에 "외부 검증: 모창 심사위원, 16년 도메인·데이터전략 설계 반영 공식 인정" 한 줄 추가 권장.

---

## 3. 재활용 자산 B — Yieldo 함정 vs 순공대장 (포지셔닝 비교표)

| Yieldo를 죽인 결함 (심사평) | 순공대장 구조 |
| --- | --- |
| 게임사마다 데이터 구조·API 상이 → **고객사별 맞춤 연동**, 온보딩 비용·확장성 부담 | 학생이 **본인 자료를 직접 업로드**(B2C/B2B2C). 고객사별 맞춤 연동 없음. 학생 430~830명 **CAC ≈ 0** 베타 채널이 온보딩·확장성 답을 이미 보유 |
| 민감한 **재무·유저 로그를 외부 솔루션에 연동** → 보안 진입 장벽 | 외부 기업 데이터를 빼오는 게 아니라 **사용자 본인 데이터가 본인 계정에** 존재. 장벽 구조 자체가 다름 |
| AI가 **capital allocation 같은 고부담 결정**을 직접 추천 → 신뢰 형성에 시간 필요 | "AI는 대신 해주는 게 아니라 사람이 하게 만든다." AI는 **"뭘 다시 풀어라" 코칭** → 결정당 리스크 낮음, 신뢰 게이트 거의 없음 |
| 대형 게임사 **내부 조직이 자체 구축** 가능 | 학생 개개인의 망각곡선 RAG는 복리 자산. 학원·학생이 자체 구축할 대상 아님 |

---

## 4. 선제 방어 — 순공대장이 Yieldo와 공유하는 단 하나의 약점

**Yieldo의 "게임사 데이터 구조 상이" → 순공대장 버전:** "다양한 사용자 자료(교재·필기·캡처)를 학습 객체로 변환하는 게 지저분하다"는 의심. SparkClaw에서 똑같이 찌를 수 있음.

**차이(= 방어 논리):**

- Yieldo = 고객사별 맞춤 연동(외주성, 확장 불가)
- 순공대장 = **온톨로지가 변환을 표준화** → 자료가 다양해도 같은 학습 객체 구조로 수렴

→ deck에서 **선제적으로 못 박을 것.** 안 박으면 심사위원이 같은 의심을 그대로 적용함.

## 5. 반면교사 — gate 구분 헷갈리지 말 것

심사위원이 Yieldo에 박은 마지막 못: *"실제 ROI 개선 효과를 얼마나 명확히 입증하느냐"* → 이건 순공대장에도 본능적으로 똑같이 물어볼 질문이지만, **investment gate(10~12월)의 영역**이다.

- **6월 28일 (application gate)**: "효과 입증" 아님. **스토리 + 채널 신뢰성**으로 답.
- **10~12월 (investment gate)**: 그때 베타의 망각률·정답률 변화로 효과 입증.

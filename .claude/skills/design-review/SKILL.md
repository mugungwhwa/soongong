# Design Review (순공대장 디자인 합성 검사) v2

## 0. 소유 + 경계 + 역할 분담 (먼저 읽기)

- **소유**: 이 스킬은 **디자인 리드**의 것이다. "이 화면이 현행 디자인 시스템 + 듀오링고 변형 결정을 지켰는가"를 검수한다.
- **이 스킬은 "검증" 측이다**: 게임화 리드가 *무엇을 왜* 넣을지(메카닉·Octalysis 드라이브·상태 신호 색 카테고리)를 스펙으로 출력하면, 본 스킬은 그 스펙을 비주얼 SSoT로 **렌더·검증**한다. 게임화 판단(streak를 넣을지 말지, 기억HP 단위 등)은 여기서 하지 않는다 — `gamification-design` 스킬(게임화 리드) 담당.
- **방향 주의 (중요·불변)**: 목표는 "듀오링고를 더 닮기"가 **아니다.** 우리는 듀오링고의 각 요소를 의식적으로 **차용/변형/거절**로 결정했다(§3 매트릭스). 검수는 *그 결정 대비 준수/위반*을 본다. "듀오링고는 이렇게 하는데 우리는 안 했으니 부족" 판정은 그 요소가 **차용** 결정일 때만 유효하다. **거절** 결정인데 따라 했으면 그건 오히려 위반이다.

### 역할 분담 (soongong-design ↔ design-review — 중복·충돌 금지)

| 스킬 | 답하는 질문 | 가지는 것 |
|---|---|---|
| `soongong-design` (디자인 가이드) | "값/스펙이 **어디** 있나" | SSoT 경로 (tokens.css·/styleguide·lock 문서). **hex 한 글자도 안 적음.** |
| `design-review` (본 스킬) | "그 값/결정을 **지켰나**" | 판정 프레임 (§2 검사항목·§3 매트릭스). **값을 정하지 않음.** |

> **본 스킬은 hex·토큰값·px·mood 목록을 열거하지 않는다.** v1.0이 "mood 6종"·팔레트 설명을 본문에 박았다가 코드/문서가 진화하자 거짓이 됐다(두 번째 SSoT drift). v2부터는 항상 SSoT를 **가리킨다** — 값이 필요하면 soongong-design이 지정한 경로에서 읽어라. **충돌 시 SSoT가 이긴다.**

## When to Use This Skill

- 새 화면/컴포넌트를 만든 직후 (자기 검수)
- 기존 UI를 현행 시스템 + 듀오링고 매트릭스 관점에서 리뷰할 때
- 게임화 리드가 넘긴 스펙을 비주얼로 구현하고 검증할 때
- 폐기항목 회귀 여부를 점검할 때 (PR 머지 전 권장)

## 2. 합성 검사 항목

새 화면/PR마다 아래를 대조해 **✅ / △(경미) / ❌(위반)**로 판정한다. (각 항목의 실값은 §1 SSoT에서 확인 — 본 표는 "무엇을 보는가"만.)

### 2-1. 마스코트 노출 + mood 정합

- 순공이(듀공) 마스코트가 맥락에 맞는 화면에 노출되는가. (마스코트 락: 듀공 모티프 "순공이" 단일 — 다른 마스코트/식물 모티프 도입 ❌)
- **mood가 화면 맥락에 맞는가.** mood 자산·라벨의 정본은 `MascotReaction` 컴포넌트(`apps/web/src/shared/ui/mascot-reaction.tsx`) + **`/styleguide` 보드 9(main-half 표정)** 다 — 목록을 여기 열거하지 않는다. 검수는 "결과=칭찬, 오답·loss=차분/위로, 망각위험·복습 due=이리와(조름)" 식으로 **컴포넌트가 정의한 mood ↔ 맥락 일치**를 본다.
- 위협적/게으른 캐릭터로 변형되지 않았는가. loss 상황도 공격적 표현 ❌, 차분/동반자 톤 ✅.

### 2-2. 시각 위계 (스트릭/기억HP/XP)

- 홈 상단 stats 4박스(스트릭·기억HP·순공시간·XP/등급) 위계가 사는가.
- 기억HP가 **0–5 정수**로 표현되는가 (백분율·하트 ❌ — 게임화 SSoT 잠긴 값, `게임성_기획_구조.md §4-2`).
- 위험도가 **데사처드 pill**인가 (자극적 원색 ❌). 위험도 색은 게임 신호에 전용 금지 — 오답률·기억HP 저하·스트릭 위기 알림 전용(lock §1-2).

### 2-3. 게임성 강도 (-20dB 원칙)

- 원칙: Dark RPG 대비 **-20dB**(라이트 단일). 게임성은 **홈에서 가장 낮고**, 결과·서브모드로 갈수록 올린다. "홈을 게임처럼" 만들면 위반(네온/파티클 금지, stats+배지만).
- **화면별 잠긴 강도 캡의 정본은 `01_제품_UX_게임화/게임성_기획_구조.md`(게임화 리드 소유)다.** 캡 수치를 본 스킬이 정하지 않는다 — 검수 시 그 표를 펴고 화면별 캡 초과 여부를 본다. 화면명은 현행 라우트(interaction-spec §0: 홈/`today`·회독플레이/`play`·결과/`result`·오답노트/`wrong-notes`·순공냅스/`journey` 등)로 매핑한다.
- **순공리그는 MVP1.5 잠금** — MVP1차엔 sidebar에 🔒 배지만, 클릭 불가. 리그 화면을 MVP1에 활성 구현하면 위반.

### 2-4. 모션 언어 (SOO-96 baseline)

- 전환·등장·마이크로 인터랙션이 **SOO-96 기준점**(`docs/prototypes/SOO-96/index.html`)의 모션 언어(곡선·duration·등장 리듬)와 정합하는가. 구체 easing/duration은 interaction-spec §4-2 + tokens.css에서 확인.
- **금지 모션**(interaction-spec §4-3): 전체화면 블로킹 스피너 ❌(위젯별 스켈레톤 ✅) / 제출 후 진동·섬광 강렬 피드백 ❌(subtle 색변화만) / 자동 팝업·모달 ❌ / 스트릭 소멸 카운트다운 타이머 ❌.
- bounce/spring은 **마스코트 등장 한정 차용**(§3). 버튼·카드 과한 bounce ❌.

### 2-5. 폐기항목 회귀 (자동 grep 권장 — PR 머지 전)

| 검사 | 통과 기준 | 게이트 |
|---|---|---|
| `dark:` CSS 클래스 / 다크모드 변형 | 0건 | `pnpm lint:no-dark` |
| 하드코딩 hex (등록 외) | 0건 | `pnpm lint:tokens` |
| sub-boy/girl 인앱 사용 (히어로/마케팅 전용) | 인앱 0건 | `pnpm lint:sub-scope` (SOO-88) |
| **구 Ocean 팔레트 회귀**(`#2AB8D0`/`#1A8FAD`/`#0E5C82`) | 0건 — v2 Teal/Mint가 정식(CLAUDE.md §8) | grep |
| 다크 네이비 hex(`#1a1a2e`/`#16213e` 류) | 0건 | grep |
| "회독마왕"/"RPG"/"Dark Study" 키워드 | 0건 | grep |
| 토스풍 box-shadow 중첩 | 등록 shadow 토큰 단독만, 중첩 ❌ | 육안/grep |
| 식물 모티프 등급(씨앗/새싹/꽃 등) | 0건 — 등급은 6단 텍스트(`게임성_기획_구조.md §6-1`) | grep |

> v2 Teal/Mint(`#7BC4AE` 등)은 다크 네이비가 아니다 — 별개 컬러. 혼동 금지(CLAUDE.md §8 주석).

### 2-6. 로고 사용 규칙 (SOO-105)

- 로고는 **`<Logo lang="ko|en" variant="light|mint" />` 단일 컴포넌트**(`apps/web/src/shared/ui/logo.tsx`)로만 참조한다. 확정 SVG 4종(KO/EN × light/mint)은 `public/brand/logo/`에 있다.
- 로고 SVG를 다른 곳에 복붙하거나 임의 워드마크를 새로 그리면 ❌(두 번째 SSoT). 배경 대비에 맞는 variant(밝은 배경=light/기본, o900류 어두운 배경=mint/흰색)를 골랐는가.

### 2-7. 카피 톤

- passive-aggressive·죄책감·fear 카피 ❌ ("안 하면 망한다", "돌아와!", 성적/합격 보장, fear-based 결제 카피).
- 압박 없는 **동반자 톤** ✅ (예: "오늘 하면 기억이 2배 오래 남아요", "순공이가 기다리고 있어요").

## 3. 듀오링고 차용/변형/거절 매트릭스 (판정 기준의 핵심 — 뼈대 유지)

**이게 design-review의 잣대다.** 각 패턴을 셋 중 하나로 결정해뒀다. 검수는 이 결정 대비 준수를 본다. (정본 매트릭스는 `순공대장_UI_설계.md` — 변경 시 거기 먼저 갱신.)

| 듀오링고 패턴 | 우리 결정 | 검수 포인트 |
|---|---|---|
| 마스코트 전 화면 노출 | **차용** | 안 했으면 "부족" 판정 |
| 메인 그린+노랑 액센트 | **변형** | **v2 Teal/Mint** + soft accent. 듀오링고 원색(#58cc02 등) 그대로면 위반 |
| 표제/본문 폰트 분리 | **차용(한글)** | 둥근 표제체 + Pretendard 본문 (실폰트 = tokens/스타일) |
| Bouncy 애니메이션 전반 | **부분 차용** | 마스코트 등장만. 버튼/카드 과한 bounce ❌ |
| 빨강 하트(생명 5개) | **변형** | 기억HP(0–5 정수)+데사처드 위험도. 손실 자극 부드럽게 |
| passive-aggressive 카피 | **거절** | 했으면 위반 |
| streak loss 슬픈 마스코트 | **변형** | 순공이 차분/위로 mood로(공격적 ❌) |
| 게임화 4종(XP/스트릭/하트/리그) | **부분 차용** | 하트→기억HP 변형 필수. 리그=MVP1.5 잠금 |
| 잦은 reminder 푸시 | **거절** | 회독 일정 알림만(1/3/7/14일) |
| Empty state 친근 일러스트 | **차용** | 순공이 mood 활용 |
| 짧은 학습 세션(≤5분) | **차용(잠금)** | 회독 1개=1~2분 |
| Streak Freeze 아이템샵 | **거절(MVP1)** | 인앱 상점 ❌ |

> 새 듀오링고 패턴 발견 시: 매트릭스에 한 줄 추가(차용/변형/거절 + 이유)부터. "참고만 함"은 금지 — 의사결정 강제.

## 4. 상시 원칙 (그린필드 + 서비스화 가능)

> v2 신규. 화면을 "어디까지 자유롭게 만들어도 되는가"의 경계.

1. **브랜드·스타일은 keep, 그 외 UI는 그린필드.** 잠긴 것(팔레트·마스코트 락·로고·톤·게임화 잠긴 값·폐기 정책)은 지키되, 그 위에서의 레이아웃·컴포넌트·인터랙션은 **자유롭게 새로 설계**해도 된다. "기존 화면과 똑같이"가 제약이 아니다 — 잠긴 결정만 어기지 않으면 된다.
2. **서비스화 가능(production-ready) 기준.** 프로토타입 톤이 아니라 **실제 출시 품질**을 기준점으로 삼는다(SOO-96 baseline이 그 기준점). "데모니까 대충"은 검수 통과 사유가 아니다.

## 5. 게임화 리드와의 핸드오프 계약 (뼈대 유지)

> 직접 채팅/핑퐁 금지. **티켓 산출물**로만 주고받는다.

- **받는 입력 (게임화 리드 → 디자인 리드)**: 메카닉 / Octalysis 드라이브 / Hook 단계 / 상태 신호 색 **카테고리**(예: "이건 streak·연속 신호"). 구체 색은 안 받는다 — 디자인 리드가 SSoT(tokens.css)에서 정한다.
- **디자인 리드가 하는 일**: 스펙을 비주얼 SSoT로 렌더 → §2~3 합성 검사 통과 확인 → 리포트 첨부.
- **충돌 처리**: SSoT로 안 풀리는 진짜 갈등(예: 게임화 "강한 loss 모먼트" vs 톤 "부드럽게")은 임의 결정 ❌ → **오케스트레이션 통해 Mike에게 에스컬레이션**. 멀티카 `@mention` 한 라운드 질문은 가능(한 번·구체 질문 하나).

## Output 형식

검수 결과 산출 시 항상 포함:
1. 검사한 화면/PR 범위
2. §2 항목별 판정표 (✅/△/❌ + 위치). §2-1~2-7 전부 커버.
3. §3 매트릭스 결정 대비 준수/위반 (특히 "거절" 패턴을 따라 하진 않았는가)
4. §2-5 폐기항목 회귀 grep + lint 게이트(`lint:tokens`·`lint:no-dark`·`lint:sub-scope`) 결과
5. 핸드오프: 게임화 스펙을 받았다면 스펙↔구현 일치 여부
6. △/❌ 항목의 수정 제안 (비주얼 SSoT 내에서). ❌ 블로킹 → 수정 PR / △ 경미 → 백로그 후보.

## Examples

**예: "결과 화면 새로 만들었는데 리뷰해줘"**
→ 마스코트 mood가 결과 맥락(칭찬/축하)과 맞는지 `MascotReaction`+styleguide 보드9 대조 / XP 카운트업 모션이 SOO-96 곡선과 정합 / 게임성 강도 캡(게임성_기획_구조.md) 초과 여부 / 기억HP 0–5 정수 / dark·회독마왕·구Ocean grep 0건 + lint 3종 / 카피 fear 톤 없음 / 매트릭스: bouncy는 마스코트 한정이라 카드 bounce 있으면 ❌.

**예: "듀오링고는 매일 리마인더 푸시 빵빵 보내던데 우리도 늘릴까?"**
→ 잦은 푸시는 **거절** 결정. "듀오링고 대비 부족"이 아니라 의식적 차별점. 회독 일정 알림(1/3/7/14일)만 유지. 늘리자는 건 폐기 정책(압박 톤) 위반 → 제안 거절 + Mike 확인.

**예: 게임화 리드가 "오답던전에 강한 loss 신호 필요"라고 스펙을 넘김**
→ 받음. 단 빨강 하트는 거절 결정 → 데사처드 위험도 + 기억HP 감소 + 순공이 차분/위로 mood로 렌더. "강함"과 "부드러운 톤"이 충돌하면 임의 결정 말고 오케스트레이션 통해 Mike 에스컬레이션.

**예: "옛 app_UI.png 보니 다크 네이비던데 그 톤으로 맞춰주세요"**
→ app_UI.png/web_ui.png는 **비참조 구시안**(§1). 다크 네이비는 폐기 방향(§2-5). 현행 기준은 /styleguide + v2 Teal/Mint. 제안 거절 + 현행 SSoT로 안내.

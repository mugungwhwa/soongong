# 패드 풀이과정 분석 플러그인

---

완전 있어. 그리고 이건 **순공대장에 반드시 넣어야 하는 기능**이야.  
문제사진만 받으면 “무슨 문제를 틀렸는지”까지 알고, **패드 풀이과정까지 받으면 “왜 틀렸는지”**를 훨씬 정확히 알 수 있어.

## 결론

순공대장에는 이 기능을 이렇게 정의하면 돼.

> **풀이과정 캡처 엔진**  
> 학생이 iPad/갤럭시탭/웹에서 문제 위에 직접 푼 손글씨 풀이를 저장하고, AI가 풀이 흐름·오답 위치·계산 실수·조건 누락을 분석해 오답회수 퀘스트로 바꾸는 기능.

기존 순공대장 데이터 설계에서도 학생 입력 원본에 **손글씨 풀이**, 풀이 반응 데이터에 **정답 여부·풀이 시간·힌트 사용·자신감**을 저장하는 방향이 이미 맞아 있어. fileciteturn2file0 또 외부 데이터 유입 엔진도 문제사진·캡처·메모·학생 풀이로그를 받아 회독 가능한 학습 객체로 바꾸는 구조라서, “패드 풀이과정”은 Source-to-Quest Engine의 핵심 입력으로 붙이면 된다. fileciteturn2file1

## 1순위 추천 구조

### MVP는 “문제 이미지 위에 풀이 캔버스”로 가면 됨

```text id="emixzo"
문제사진 업로드
→ 문제 이미지가 배경으로 깔림
→ 학생이 Apple Pencil / S펜 / 손가락으로 풀이
→ 풀이 stroke 저장
→ 정답 제출
→ AI가 문제 + 풀이과정 + 정오답 분석
→ 오답 원인 태깅
→ 오답회수 모드 생성
```

여기서 중요한 건 풀이를 그냥 이미지로 저장하지 말고, 가능하면 **stroke 데이터**로 저장하는 거야.

```json id="6rigto"
{
  "attempt_id": "attempt_001",
  "question_id": "q_001",
  "strokes": [
    {
      "points": [
        {"x": 120, "y": 340, "t": 0, "pressure": 0.42},
        {"x": 124, "y": 343, "t": 16, "pressure": 0.45}
      ],
      "tool": "pen"
    }
  ],
  "solve_time_sec": 184,
  "eraser_count": 3,
  "submitted_answer": "3",
  "is_correct": false
}
```

이렇게 저장하면 AI가 나중에 “최종 답”뿐 아니라 **풀이 흐름**을 볼 수 있어.

## 쓸만한 오픈소스 / SDK 후보

| 용도 | 후보 | 판단 |
|---|---|---|
| 웹 캔버스 MVP | **Excalidraw** | Next.js에 빨리 붙이기 좋음 |
| 웹 고급 캔버스 | **Konva / react-konva** | 문제 이미지 위 stroke/도형/레이어 직접 구현에 좋음 |
| 웹 제품형 무한캔버스 | **tldraw** | 훌륭하지만 상업용 프로덕션은 라이선스 확인 필요 |
| 앱 캔버스 | **React Native Skia** | Expo/React Native와 잘 맞음 |
| iPad 네이티브 최상 | **Apple PencilKit** | iPad 필기감 최고, 단 Expo면 네이티브 브릿지 필요 |
| Android/S펜 | **Android MotionEvent / Ink API 계열** | 압력·기울기·hover·palm 관련 데이터 활용 가능 |
| 손글씨 인식 | **Google ML Kit Digital Ink** | stroke 기반 필기 인식, 오프라인 가능 |
| 수학식/필기 인식 | **MyScript iink SDK** | 수학식·도형·텍스트 인식 쪽 강력, 상용 SDK 검토 |

Excalidraw는 React 앱에 직접 임베드할 수 있고, Next.js에서는 SSR을 끄고 client component로 붙이는 방식이 공식 문서에 나와 있어 웹 MVP에 빠르다. ([docs.excalidraw.com](https://docs.excalidraw.com/docs/%40excalidraw/excalidraw/integration)) Konva/react-konva는 React 방식으로 Canvas 그래픽을 다루는 라이브러리이고, 브라우저 전용이라 웹 풀이판 구현에 적합하다. ([konvajs.org](https://konvajs.org/docs/react/index.html))

다만 tldraw는 제품 품질은 좋은데, SDK가 기본 조건상 개발용만 허용되고 프로덕션 사용에는 trial/commercial/hobby license가 필요하다고 공식 문서에 명시되어 있어 상용 앱이면 비용을 봐야 한다. ([tldraw.dev](https://tldraw.dev/community/license))

앱 쪽은 Expo React Native로 간다면 `@shopify/react-native-skia`가 현실적이야. Expo 문서에서도 Skia는 Android, iOS, web을 지원하는 React Native 그래픽 라이브러리로 안내되어 있어. ([docs.expo.dev](https://docs.expo.dev/versions/latest/sdk/skia/))

## iPad 최적화는 PencilKit이 제일 좋음

iPad 학생을 제대로 잡으려면 장기적으로는 **Apple PencilKit**이 가장 좋아. Apple은 PencilKit을 iPad 앱에서 쓰기·그리기·애니메이션을 지원하는 drawing/annotation framework로 설명하고, PKCanvasView, PKToolPicker, PKStroke 같은 API를 제공한다고 안내해. ([developer.apple.com](https://developer.apple.com/videos/play/wwdc2020/10107/))

다만 순공대장 앱을 Expo React Native로 만든다면 PencilKit은 바로 붙는 게 아니라, **Swift 네이티브 모듈을 브릿지**해야 해. 그래서 순서는 이렇게 추천.

```text id="ordul2"
MVP:
React Native Skia로 자체 풀이 캔버스 구현

iPad 고도화:
PencilKit 네이티브 모듈 추가

Android 고도화:
S펜 압력/기울기/hover/palm rejection 데이터 추가
```

Android도 스타일러스 입력에서 pressure, orientation, tilt, hover, palm detection 같은 정보를 MotionEvent로 다룰 수 있다고 공식 문서가 설명해. ([developer.android.com](https://developer.android.com/develop/ui/compose/touch-input/stylus-input/advanced-stylus-features))

## 손글씨 인식은 어디까지 가능하냐

### 1. 일반 필기 인식

Google ML Kit Digital Ink Recognition은 사진 OCR이 아니라 **화면에 쓴 stroke 데이터**를 인식하는 방식이야. 공식 문서에 따르면 필기 텍스트와 제스처를 인식하고, 300개 이상 언어와 25개 이상 writing system을 지원하며, Android/iOS에서 오프라인으로 작동한다. ([developers.google.com](https://developers.google.com/ml-kit/vision/digital-ink-recognition))

### 2. 수학식 인식

수학 풀이까지 제대로 하려면 MyScript 쪽이 더 강함. MyScript Interactive Ink SDK 예제는 손글씨 텍스트, 수학식, 다이어그램을 실시간으로 해석해 편집 가능한 출력으로 바꿀 수 있다고 설명하고, Math content type 예제도 제공한다. ([github.com](https://github.com/MyScript/interactive-ink-additional-examples-android))

즉 판단은 이거야.

| 목표 | 추천 |
|---|---|
| “학생이 어디에 뭘 썼는지” 저장 | 자체 stroke 저장 |
| 손글씨 텍스트 일부 인식 | Google ML Kit Digital Ink |
| 수학식 LaTeX 변환 | MyScript iink SDK |
| 풀이과정 전체 의미 분석 | stroke + 이미지 렌더링 + LLM/Vision 분석 조합 |

## MVP 구현 우선순위

| 단계 | 기능 | 기술 |
|---|---|---|
| 1 | 문제 이미지 위에 필기 | React Native Skia / 웹은 Konva |
| 2 | stroke JSON 저장 | Supabase PostgreSQL + Storage |
| 3 | 풀이 PNG 렌더링 저장 | Storage 임시 저장 |
| 4 | 풀이 시간·지우개 횟수·힌트 사용 저장 | solve_events |
| 5 | AI가 풀이 이미지 분석 | Vision LLM |
| 6 | 오답 원인 태깅 | 오답진단 Agent |
| 7 | 오답회수 문제 생성 | Variation Agent |
| 8 | iPad PencilKit 고도화 | 네이티브 모듈 |

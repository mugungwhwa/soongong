# 앱 포팅 기술 추천

> 원본 소스: `앱 포팅 기술 추천.txt`  
> 분류: **개발 스택·구현**

결론: **앱은 Expo React Native로 가는 게 맞아.**  
웹이 **Next.js + FSD 2.1**이면, 앱은 **Expo Router + React Native + FSD 변형 구조**로 가져가면 된다.

순공대장은 앱에서 **오늘의 회독, 카메라 업로드, 푸시 알림, 스트릭, 회독 플레이**가 핵심이고, 웹은 **대시보드·업로드·리포트·관리 화면** 역할이 강해. 기존 UI/기능 설계에서도 앱은 매일 회독 중심, 웹은 기록 관리·업로드·리포트 역할로 분리하는 방향이 맞다고 잡혀 있어. fileciteturn3file0

## 왜 Expo React Native냐

순공대장은 앱에서 네이티브 기능이 꽤 중요해.

| 기능 | 중요도 | 이유 |
|---|---:|---|
| 카메라 촬영 | 매우 높음 | 문제사진 업로드가 핵심 |
| 이미지 업로드 | 매우 높음 | 오답/캡처 저장 |
| 푸시 알림 | 매우 높음 | 망각방어, 스트릭 리텐션 |
| 앱 위젯 | 중기 | 오늘 회독 알림 |
| 오프라인 저장 | 중기 | 지하철/독서실 환경 |
| 앱스토어 배포 | 높음 | B2C 수능 앱 신뢰감 |

Expo는 React Native 기반으로 카메라, 이미지, 푸시, 빌드/배포 흐름을 비교적 빠르게 잡을 수 있고, React/TypeScript 기반이라 Next.js 웹과 사고방식을 맞추기 좋다. Expo 공식 문서도 React Native가 JavaScript 코드에서 Camera, Notifications 같은 네이티브 API에 접근하게 해주는 구조라고 설명한다. ([docs.expo.dev](https://docs.expo.dev/develop/development-builds/introduction/?utm_source=chatgpt.com)) Expo Notifications도 Android/iOS 푸시를 다루는 통합 API를 제공한다. ([docs.expo.dev](https://docs.expo.dev/guides/using-push-notifications-services/?utm_source=chatgpt.com))

## 비추천: Next 웹을 그대로 WebView로 감싸기

초기 데모만 빨리 낼 때는 가능하지만, 순공대장 메인 앱으로는 비추야.

```txt id="ry2imx"
Next.js WebView 앱
장점: 빠름
단점: 앱 같은 느낌 약함, 카메라/푸시/오프라인/성능/스토어 품질에서 한계
```

순공대장은 듀오링고형 앱처럼 매일 들어와서 회독을 깨는 제품이라, **앱 체감 품질**이 중요해. WebView로 가면 “웹서비스를 앱처럼 포장한 느낌”이 날 가능성이 커.

## Flutter는?

Flutter도 가능하지만 지금 구조에서는 우선순위가 낮아.

| 선택지 | 판단 |
|---|---|
| **Expo React Native** | 추천 |
| Flutter | 디자인/앱 완성도는 좋지만 웹 Next와 코드 공유가 거의 안 됨 |
| Capacitor | 초초기 래핑용 가능, 장기 메인앱은 애매 |
| Native iOS/Android | 너무 무거움 |
| PWA | 앱스토어/푸시/카메라 경험이 약함 |

웹을 Next.js로 이미 정했다면, Flutter로 앱을 따로 만들면 **프론트 스택이 2개**가 돼. 1인/소규모 개발에서는 유지보수가 커진다.

## FSD를 앱에 적용하는 방식

웹 FSD를 앱에 그대로 복붙하지 말고, **개념만 맞추면 돼.**

```txt id="dg7v7q"
/apps/mobile/src
  /app              앱 provider, navigation setup
  /pages            화면 단위
    /today
    /upload
    /review-play
    /result
    /stats
    /profile

  /widgets          여러 feature 조합
    /today-review-path
    /quest-card-list
    /bottom-tabs
    /haemawang-coach

  /features         사용자 행동 단위
    /upload-source
    /start-review
    /submit-answer
    /complete-quest
    /schedule-review

  /entities         도메인 객체
    /user
    /quest
    /source
    /question
    /mastery
    /streak

  /shared           공통
    /api
    /ui
    /config
    /lib
    /types
```

Expo Router는 `/app` 폴더를 라우팅으로 쓰니까, 실제 라우트 파일은 얇게 두면 돼.

```txt id="pfr0ks"
/apps/mobile/app/(tabs)/today.tsx
→ src/pages/today/ui/TodayPage.tsx import
```

이렇게 하면 Expo Router와 FSD를 충돌 없이 같이 쓸 수 있어.

## 한 줄 결정

> **웹은 Next.js + FSD 2.1, 앱은 Expo React Native + Expo Router + FSD 변형 구조.**  
> 웹을 앱으로 감싸는 게 아니라, **도메인/타입/API/디자인 토큰은 공유하고 UI는 앱답게 재구현**하는 게 맞아.

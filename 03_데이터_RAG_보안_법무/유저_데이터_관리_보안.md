## 순공대장 유저 데이터 관리 및 보안 v0.1

순공대장은 일반 학습앱보다 데이터 리스크가 큽니다. 이유는 **미성년자 가능성, 문제사진, 인강 캡처, 손글씨 풀이, 학습능력 진단, AI 생성 로그**가 모두 섞이기 때문입니다. 그래서 핵심 원칙은 하나로 잡으면 됩니다.

> **원본 데이터는 최소 저장하고, 파생 학습 데이터만 안전하게 누적한다.**

즉, 순공대장은 “문제 원문/인강 캡처를 많이 저장하는 앱”이 아니라, 이미 잡아둔 것처럼 **문제사진·인강기록·캡처·메모·풀이로그를 받아 회독 가능한 학습 객체로 정제하는 Source-to-Quest Engine**이어야 합니다. 내부 설계에서도 Source Registry → Raw Intake Zone → Compliance Gate → Learning Object Builder → RAG Indexer → Quest Publisher 흐름으로 잡혀 있습니다. fileciteturn2file0

# 1. 데이터 분류

순공대장 데이터는 이렇게 7개로 나눠야 합니다.

| 데이터 구분 | 예시 | 저장 정책 |
|---|---|---|
| 계정 데이터 | 이름, 이메일, 전화번호, 학년 | 최소 수집 |
| 학생 입력 원본 | 문제사진, 인강 캡처, 손글씨 풀이 | 임시/제한 저장 |
| 파생 학습 데이터 | 과목, 단원, 유형, 오답 원인, 난이도 | 장기 저장 가능 |
| 풀이 반응 데이터 | 정답 여부, 풀이 시간, 힌트 사용, 자신감 | 개인화 핵심 데이터 |
| AI 분석 데이터 | OCR 결과, 태깅 결과, 생성 문제, 해설 | 검수/버전 관리 필요 |
| 게임 데이터 | XP, 스트릭, 리그 점수, 기억 HP | 장기 저장 가능 |
| 운영/보안 로그 | 관리자 접근, 오류 신고, Agent 실행 로그 | 감사 로그로 보관 |

가장 중요한 분리는 이것입니다.

| 구분 | 의미 | 정책 |
|---|---|---|
| **Raw Data** | 원본 이미지, 문제 원문, 인강 캡처 | 최소 저장, 자동 삭제 |
| **Derived Data** | 과목, 단원, 유형, 오답 원인, 복습카드 | 서비스 핵심 자산으로 저장 |

기존 설계에서도 “Raw Data = 원본 이미지/원문/캡처, Derived Data = 과목·단원·유형·오답원인·난이도·복습카드·유형카드”로 분리하고, 순공대장은 Raw를 많이 쌓는 서비스가 아니라 Derived를 쌓는 서비스여야 한다고 정리되어 있습니다. fileciteturn2file0

# 2. 핵심 보안 원칙

## 2-1. 원본 문제사진/캡처는 오래 들고 있지 않는다

추천 정책은 이겁니다.

| 원본 유형 | 정책 |
|---|---|
| 문제사진 | OCR/분석 후 일정 기간 보관, 이후 삭제 또는 비식별화 |
| 인강 캡처 | 원본 장기 저장 지양, 파생 데이터만 저장 |
| 유료 인강 영상 | 저장 금지 |
| 기출 원문 | 서비스 노출 금지 방향 |
| 손글씨 풀이 | 사용자별 비공개 저장, 공유 금지 |
| 제휴 문제 | 계약 범위 내 저장/노출 |

특히 인강 캡처는 **원본 이미지를 장기 저장하지 않고, OCR/개념태그/학생메모 기반 파생 데이터만 저장**하는 방식이 안전합니다. 기존 법무/기능 설계에서도 인강 전체 녹화, 자막 전체 추출, 강의 화면 자동 캡처, 강의 영상 저장, 강의자료 공유는 위험한 설계로 정리되어 있습니다. fileciteturn2file1

# 4. 접근 권한 설계

Supabase를 쓴다면 반드시 **RLS(Row Level Security)**를 켜야 합니다. Supabase 공식 문서도 RLS를 Postgres의 방어 계층으로 설명하고, Supabase Auth와 결합해 브라우저에서 DB까지 사용자별 접근 제어를 구현할 수 있다고 안내합니다. ([supabase.com](https://supabase.com/docs/guides/database/postgres/row-level-security?utm_source=chatgpt.com))

| 역할 | 접근 가능 데이터 |
|---|---|
| 학생 | 본인 데이터만 |
| 보호자 | 연결된 학생의 요약 리포트만 |
| 검수자 | 원본이 아닌 검수 대상 학습 객체 중심 |
| 관리자 | 운영상 필요한 최소 범위 |
| AI Agent | 처리 목적별 제한된 테이블만 |
| 개발자 | 운영 DB 직접 접근 금지, 로그/샘플만 |

Storage도 RLS 기반 접근 정책을 걸 수 있으므로, 문제사진/캡처는 사용자별 private bucket 또는 path 단위 정책으로 제한해야 합니다. Supabase Storage는 Postgres RLS와 함께 접근 정책을 만들 수 있다고 공식 문서가 설명합니다. ([supabase.com](https://supabase.com/docs/guides/storage/security/access-control?utm_source=chatgpt.com))

# 5. 개인정보/청소년 정책

수능생 대상이면 대부분 만 14세 이상이지만, 중학생 선행학습 유저가 들어올 수 있습니다. 개인정보보호법은 만 14세 미만 아동의 개인정보 처리에 동의가 필요한 경우 법정대리인의 동의를 받아야 하고, 고지는 이해하기 쉬운 양식과 명확한 언어를 사용해야 한다고 규정합니다. ([law.go.kr](https://www.law.go.kr/LSW//lsSideInfoP.do?docCls=jo&joBrNo=02&joNo=0022&lsiSeq=270351&urlMode=lsScJoRltInfoR&utm_source=chatgpt.com))

추천 정책은 이겁니다.

| 항목 | 정책 |
|---|---|
| 가입 연령 | 초기 MVP는 만 14세 이상만 허용 권장 |
| 만 14세 미만 | 보호자 동의 플로우 없으면 가입 제한 |
| 개인정보처리방침 | 필수 |
| 아동/청소년 안내문 | 쉬운 버전 별도 제공 권장 |
| 데이터 삭제권 | 앱 내 탈퇴/삭제 요청 제공 |
| AI 학습 사용 | 기본 비동의, 별도 opt-in |
| 제3자 처리위탁 | OCR, LLM, 클라우드 업체 명시 |
| 국외 이전 | 해외 API 사용 시 고지 필요 |
| 보관기간 | 원본/파생/로그별로 다르게 설정 |

개인정보보호위원회는 개인정보 처리방침 작성지침을 제공하고 있으며, 처리 목적, 항목, 보유기간, 파기, 위탁, 국외 이전, 안전성 확보조치 등을 포함해 투명하게 작성하도록 안내합니다. ([privacy.go.kr](https://www.privacy.go.kr/front/bbs/bbsView.do?bbsNo=BBSMSTR_000000000049&bbscttNo=20806&utm_source=chatgpt.com))

# 7. 보관 기간 정책

MVP에서는 이렇게 잡으면 좋습니다.

| 데이터 | 보관 기간 |
|---|---|
| 원본 문제사진 | 7~30일 후 자동 삭제 또는 사용자가 선택 |
| 인강 캡처 원본 | 24시간~7일 임시 보관 권장 |
| OCR 텍스트 | 사용자 학습 객체로 필요한 범위만 보관 |
| 파생 학습 객체 | 계정 유지 기간 동안 보관 |
| 풀이 반응 데이터 | 계정 유지 기간 동안 보관 |
| AI Agent 로그 | 90~180일 |
| 감사 로그 | 1~3년 |
| 결제/환불 기록 | 관련 법령 기준 별도 보관 |
| 탈퇴 사용자 데이터 | 즉시 삭제 또는 법정 보관분 분리 저장 |

핵심은 **원본과 파생 데이터를 같은 보관기간으로 두면 안 된다**는 점입니다.

# 10. 사업계획서용 문장

바로 넣으려면 이렇게 쓰면 됩니다.

> 순공대장은 수능생의 문제사진, 인강 기록, 캡처, 풀이 반응을 수집하되 원본 데이터와 파생 학습 데이터를 분리해 관리한다. 문제사진·인강 캡처 등 원본 데이터는 OCR 및 AI 분석 후 일정 기간 내 삭제하거나 비식별화하고, 서비스 운영에는 과목·단원·유형·오답 원인·복습 일정 등 파생 학습 객체를 중심으로 활용한다. 사용자별 데이터는 Row Level Security 기반으로 분리하고, 관리자·검수자·AI Agent의 접근 권한을 최소화하며, 모든 접근과 생성 결과는 감사 로그로 기록한다. 또한 만 14세 미만 이용자 보호, 개인정보 처리방침, 제3자 처리위탁·국외 이전 고지, AI 생성물 고지, 오류 신고 및 데이터 삭제권을 제공해 교육 서비스에 필요한 개인정보 보호와 신뢰성 기준을 충족한다.

## 1. 창업자 / 사업 토대

### 1.2 창업자 자산 (Founder Assets)

순공대장의 모트는 코드가 아니라 **창업자의 의사결정 설계 능력**에 있다.

- **도메인 근육**: 게임 산업 16년 — 마케팅 · 데이터 전략 · 리텐션. "불편한 걸 매일 자발적으로 견디게 만드는 일"(회독/인출 = desirable difficulty)은 교육학이 아니라 리텐션 루프 문제이며, 이게 Mike가 적임인 이유. (출처: `2026-06-08-순공대장_SparkClaw_1기_포지셔닝_보강.md`)
- **희소 자산 = DA 의사결정 설계**: prior 설계, A/B 분산 축소, retention leading indicator 판별, 망각곡선 파라미터 추정 — 마케팅의 CAC/LTV/퍼널 본능이 그대로 이식된 것. 데이터 *엔지니어링*이 아니라 *판단 체계*가 모트.
- **Founder authenticity**: 콴다 창업자 = 잘하는 사람의 도구(서울대 수학과). Mike = 회독이 안 돼 무너져 본 사람(대부분의 문제). "막힘 vs 까먹음" 포지셔닝이 여기서 나온다.
- **외부 검증 (제3자)**: 모창(Yieldo Terminal 심사) 심사위원이 **16년 도메인·데이터 전략이 서비스 설계 전반에 반영**됐고, 운영·실험·재무·시장을 하나의 판단 체계로 통합하려는 접근(베이지안/A·B 사고)을 **공식 인정**. → SparkClaw application gate가 요구하는 "데이터 사고 + 빌더 신뢰성"을 제3자가 문서로 확인해준 셈.
  - 상세 / 재활용 자산(포지셔닝 비교표 · 선제 방어 논리 · gate 구분): **`2026-06-09-yieldo-심사평-sparkclaw-재활용자산.md`**

### 2.3 내러티브 레이어 정책 (2026-06-10 확정)

> "수험생 전용 vs 글로벌 학습 비전"은 양자택일이 아니라 **레이어**다.
> 노출면마다 아래 단을 고정 사용한다. 단을 섞거나 건너뛰는 제안은 거절하고 Mike에게 확인.

| 레이어 | 메시지 | 사용처 |
| --- | --- | --- |
| **실행** | 한국 수능. 좁을수록 좋다 | 제품 · README · 베타 운영 · 학원 채널 |
| **내러티브** | "고부담 시험의 기억 문제를 푸는 **엔진** — 세계 최고 밀도 시험 시장(한국 수능)에서 검증 시작." 확장 사다리(수능→공시→전문직→일본→미국)는 한 슬라이드 | SparkClaw deck · 1000자 자기소개 · VC 미팅 |
| **정체성** | "모든 지식 획득 여정의 동반자" + v1.0~v5.0 진화 경로 | PROJECT.md · 내부 나침반. **피치 재료 아님** |

- v5.0 "전지식 동반자" 레벨의 비전은 traction이 받쳐줄 때(시리즈 A 이후)까지 대외 노출 금지. 베타 0명 상태에서 가장 공격받기 쉬운 표적이 된다.
- 반대로 deck이 수능에만 갇히는 것도 금지 (valuation 6~8x 함정). 6/28 정답 = **"엔진은 글로벌, 비치헤드는 수능."**

### 2.4 모델 · 트랙 통제 원칙 (2026-06-10 확정)

**트랙 분리 (혼동 금지):**

| 트랙 | 실체 | 트리거 |
| --- | --- | --- |
| 개발 트랙 | 둥지(Multica) — Orchestration/Tech Lead/도메인 리드 | Mike가 일할 때 |
| 서비스 트랙 | 제품 코드 안의 파이프라인 16 (Claude API 호출 단계) | 학생이 앱을 쓸 때 |

- **서비스 런타임에 Multica/외부 오케스트레이터 연결 금지** (레이턴시·비용·장애점·통제 문제)
- 학생 상태 추정 = **통계 모델**. LLM은 제안자/피처 추출기(오답원인 태깅 등)이지 최종 판정자가 아니다

**모델 로드맵 (단계 전환은 Mike 결정):**

| 단계 | 모델 | 시점 |
| --- | --- | --- |
| 1 | 고정 간격 1/3/7/14 + 단순 망각위험 점수 (현 MVP) | ~6/28 그대로 |
| 2 | FSRS (스케줄링) + Elo (학생 능력 × 문항 난이도 온라인 추정) | 베타 7~10월 |
| 3 | 계층 베이지안 IRT (온톨로지 = prior 구조 + Q-matrix) | 시리즈 A 전후, 수만 응답 누적 이후 |

- 생성 문항(V0~V5) 난이도 = **부모 문항 난이도 + 변형축 오프셋을 prior로 상속** → 첫 20~50 응답으로 온라인 보정. LLM 난이도 판정은 초기값일 뿐, 실제 정답률이 항상 우선
- 모트는 모델이 아니라 **모델에 먹이는 판단** (온톨로지 구조 · prior · 오프셋 · 수락 기준). FSRS/Elo 등 검증된 외부 모델 채택은 모트 훼손이 아니다

**노브 원칙 (비개발자 통제권의 물리적 실체):**

- Mike가 조정하는 값(prior 초기값 · 변형축 오프셋 · 캘리브레이션 수락 임계치 · 회독 간격 상하한)은 코드에 묻지 않고 **별도 설정 파일/테이블로 분리**한다
- 해당 설정 파일 변경은 Mike 승인 없이 머지 금지 — 이 파일이 MOAT 레이어의 git상 실체

**품질 게이트:**

- 모델 품질 = **캘리브레이션**으로 측정 (Brier score / log-loss, 예측 정답률 vs 실제 정답률). 수락 기준 미달 시 배포 불가. 기준 숫자는 Mike가 결정 (비위임)
- **평가자 에이전트** (둥지 9번째, 베타 전 등록 예정): 주간 캘리브레이션 리포트만 출력, 기준 미달 시 플래그 후 정지. 모델 자율 수정 금지 — 유형 리서치 에이전트와 동일한 "제안/측정만 출력" 패턴
- 위임 경계: 에이전트 = 모델 코드 · 파이프라인 · 측정 하네스 · 리포트 / Mike = 골드셋 큐레이션 · 수락 기준 · 설정값 승인 · 모델 단계 전환

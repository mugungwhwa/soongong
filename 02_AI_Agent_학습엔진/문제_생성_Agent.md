# 문제 생성 Agent

> 원본 소스: `문제 생성 Agent.txt`  
> 분류: **AI Agent·학습 엔진**

---

맞아. **문제 유형 베리에이션**은 순공대장의 핵심 엔진 중 하나야.  
여기서 중요한 건 “비슷한 문제 많이 만들기”가 아니라,

> **같은 개념을 다른 형태로 다시 만나게 만드는 것**

이야.

즉, 베리에이션 Agent는 문제를 이렇게 다뤄야 해.

> 원문 문제 → 문항 DNA 추출 → 변형 축 선택 → 난이도 조절 → 오답 원인 반영 → 검증 → 출제

---

# 1. 문제 유형 베리에이션은 “문항 DNA”로 쪼개야 한다

문제 하나를 그대로 변형하면 안 돼.  
먼저 문제를 아래처럼 분해해야 해.

## 문항 DNA 구조

| 요소 | 의미 | 예시 |
|---|---|---|
| 핵심 개념 | 무엇을 묻는가 | 점화식, 등차수열, 조건부확률 |
| 풀이 전략 | 어떻게 풀어야 하는가 | 반복 대입, 식 변형, 경우 나누기 |
| 표현 형식 | 어떤 형태로 제시되는가 | 식, 표, 그래프, 지문, 상황문 |
| 요구값 | 무엇을 구하라는가 | 일반항, 특정 항, 최댓값, 옳은 선지 |
| 조건 구조 | 어떤 조건이 붙는가 | 초항, 범위, 단서, 예외 조건 |
| 오답 유도 | 어디서 틀리게 만드는가 | 조건 누락, 계산 실수, 개념 혼동 |
| 난이도 요소 | 왜 어려운가 | 계산량, 조건 수, 개념 결합 수 |

예를 들어 수열 문제라면 이렇게 쪼갤 수 있어.

```json id="0s1bwp"
{
  "concept": "수열_점화식",
  "strategy": "반복 대입 후 패턴 추론",
  "representation": "수식형",
  "target": "a_n 일반항",
  "conditions": ["초항 제공", "점화식 제공"],
  "common_mistake": ["n/n+1 혼동", "초항 누락"],
  "difficulty_factors": ["풀이 단계 3개", "계산량 중간"]
}
```

이렇게 해두면 AI가 원문을 베끼지 않고 **구조만 활용해서 새 문제를 만들 수 있어.**

---

# 2. 베리에이션은 6단계로 만든다

처음부터 멀리 변형하면 학생이 못 따라와.  
순공대장은 회독 앱이니까 “가까운 변형 → 먼 변형”으로 가야 해.

## Variation Level

| 단계 | 이름 | 설명 | 목적 |
|---:|---|---|---|
| V0 | 원문 회독 | 학생이 올린 문제 그대로 다시 풀기 | 기억 회수 |
| V1 | 숫자/조건 미세변형 | 숫자, 초항, 계수만 변경 | 같은 풀이 재훈련 |
| V2 | 요구값 변경 | 일반항 → 특정 항, 값 비교 등 | 풀이 이해 확인 |
| V3 | 표현 변경 | 식 → 표, 그래프, 상황문 | 개념 전이 |
| V4 | 조건 구조 변경 | 조건 추가/삭제/역방향 질문 | 수능형 적응 |
| V5 | 개념 결합 | 수열+함수, 확률+경우의 수 등 | 4점보스 |

이게 핵심이야.

> 학생이 방금 틀렸으면 V1~V2  
> 어느 정도 맞추면 V3~V4  
> 숙련도가 높으면 V5

---

# 3. 베리에이션 축은 이렇게 잡는다

문제 유형을 다양하게 만드는 축은 크게 8개야.

| 변형 축 | 예시 |
|---|---|
| 숫자 변형 | 계수, 초항, 범위 변경 |
| 조건 변형 | 조건 추가, 조건 삭제, 조건 순서 변경 |
| 요구값 변형 | 값을 구하라 → 옳은 것을 고르라 |
| 표현 변형 | 식 → 표 → 그래프 → 상황문 |
| 선지 변형 | 객관식, 단답형, OX, 빈칸 |
| 오답 원인 변형 | 계산 실수 유도 → 조건 누락 유도 |
| 난이도 변형 | 풀이 단계 수, 개념 수, 계산량 조정 |
| 맥락 변형 | 추상 수식 → 실생활 상황형 |

예를 들어 같은 점화식 개념도 이렇게 변형돼.

| 유형 | 예시 |
|---|---|
| V1 숫자 변형 | `a₁=1, aₙ₊₁=aₙ+2n` |
| V2 요구값 변형 | `a₅의 값을 구하라` |
| V3 표현 변형 | 표로 `a₁, a₂, a₃` 일부를 제시 |
| V4 조건 변형 | `a₃=7일 때 a₁을 구하라` |
| V5 결합 변형 | 수열 조건과 함수값 조건을 함께 제시 |

---

# 4. 수학 예시: 점화식 문제 베리에이션

원래 학생이 틀린 개념이 이거라고 해보자.

> 개념: 점화식  
> 오답 원인: n과 n+1 인덱스 혼동  
> 현재 숙련도: 62점  
> 추천 난이도: L2~L3

그러면 베리에이션은 이렇게 나가면 돼.

## V1: 가까운 변형

```text id="g40icl"
a₁=2, aₙ₊₁=aₙ+3일 때 a₄의 값은?
```

목적: 점화식의 기본 진행 확인.

---

## V2: 요구값 변경

```text id="7npep0"
a₁=1, aₙ₊₁=aₙ+2n일 때 a₃의 값은?
```

목적: n에 어떤 값을 넣어야 하는지 확인.

---

## V3: 표현 변경

```text id="ufrttg"
다음 표는 수열 {aₙ}의 일부이다.

n      1   2   3   4
aₙ     1   3   7   ?

aₙ₊₁ - aₙ = 2n일 때 a₄의 값은?
```

목적: 식이 아니라 표로 봐도 같은 구조를 인식하는지 확인.

---

## V4: 조건 역방향

```text id="ol0oih"
수열 {aₙ}이 aₙ₊₁=aₙ+2n을 만족하고 a₃=7이다. a₁의 값은?
```

목적: 앞으로만 대입하는 게 아니라 역추적 가능 여부 확인.

---

## V5: 4점보스형

```text id="obtptp"
수열 {aₙ}이 a₁=1, aₙ₊₁=aₙ+2n을 만족한다. 
자연수 k에 대하여 a_k < 50 ≤ a_{k+1}일 때 k의 값은?
```

목적: 점화식 + 부등식 판단 결합.

이렇게 하면 같은 개념을 계속 반복하지만, 학생 입장에서는 “복붙 문제”가 아니라 **점점 깊어지는 회독**처럼 느껴져.

---

# 5. 베리에이션 Agent는 학생 상태에 따라 다르게 출제해야 한다

베리에이션을 랜덤으로 뽑으면 안 돼.  
학생 상태에 따라 변형 강도를 조절해야 해.

| 학생 반응 | 다음 베리에이션 |
|---|---|
| 방금 오답 | V0 또는 V1 |
| 오답 후 해설 봄 | V1 + 개념카드 |
| 정답이지만 오래 걸림 | V2 |
| 빠른 정답 | V3 |
| 3회 연속 정답 | V4 |
| 해당 개념 숙련도 80 이상 | V5 보스전 |
| 같은 오답 원인 반복 | 같은 난이도, 오답 원인만 바꾼 변형 |

즉, 베리에이션 정책은 이렇게 돼야 해.

```text id="dr0cwg"
낮은 숙련도 → 가까운 변형
중간 숙련도 → 표현/요구값 변형
높은 숙련도 → 조건/개념 결합 변형
```

---

# 6. 오답 원인별 베리에이션도 달라야 한다

오답 원인이 다르면 변형 방식도 달라야 해.

| 오답 원인 | 좋은 베리에이션 |
|---|---|
| 개념 미흡 | OX, 빈칸, L1 개념 회수 |
| 계산 실수 | 숫자만 바꾼 짧은 재계산 |
| 조건 누락 | 조건 찾기형 문제 |
| 인덱스 혼동 | n/n+1 비교형 문제 |
| 선지 과잉해석 | 근거 찾기형 문제 |
| 시간 부족 | 풀이 단계 분해형 문제 |

예를 들어 학생이 “조건 누락”으로 틀렸다면 단순히 쉬운 문제를 주는 게 아니라, 이렇게 줘야 해.

```text id="8g87uh"
다음 문제에서 반드시 사용해야 하는 조건을 모두 고르시오.
```

이건 문제풀이가 아니라 **오답 원인을 직접 치료하는 베리에이션**이야.

# 7. 문제 형식 자체도 바꿔야 한다

순공대장은 매번 객관식 5지선다만 내면 지루해져.  
회독용 문제는 다양한 형태로 나눠야 해.

| 형식 | 목적 |
|---|---|
| OX | 개념 회수 |
| 빈칸 | 공식/조건 기억 |
| 단답형 | 계산 확인 |
| 객관식 | 수능형 적응 |
| 조건 찾기 | 조건 누락 방지 |
| 오류 찾기 | 오답 원인 교정 |
| 풀이 순서 배열 | 풀이 전략 훈련 |
| 미니 보스 | 실전형 도전 |

예를 들어 수열 점화식 하나로도:

```text id="5eqqu7"
OX: 점화식 문제에서는 초항 확인이 필요 없다. O/X

빈칸: aₙ₊₁=aₙ+2n에서 a₃를 구하려면 n=__까지 대입한다.

오류 찾기: 다음 풀이에서 잘못된 부분을 고르시오.

객관식: a₁=1, aₙ₊₁=aₙ+2n일 때 a₄는?

풀이 순서: 점화식 문제 풀이 순서를 바르게 배열하시오.
```

이게 듀오링고형 학습에 가까워.  
긴 문제만 계속 던지면 피로도가 높아져.

# 8. 과목별 베리에이션 전략

## 수학

수학은 가장 구조화하기 좋다.

| 변형 축 | 예시 |
|---|---|
| 숫자 | 계수, 초항 변경 |
| 조건 | 범위, 부등식 조건 추가 |
| 요구값 | 특정값, 일반항, 최대/최소 |
| 표현 | 식, 표, 그래프 |
| 결합 | 함수, 확률, 도형과 결합 |

수학은 **템플릿 + 자동 검증**으로 갈 수 있어.

---

## 영어

영어는 원문 지문 저작권 리스크가 있으니까 조심해야 해.  
초기에는 긴 지문 생성보다 짧은 유형으로 가는 게 좋아.

| 변형 유형 | 예시 |
|---|---|
| 어휘 회독 | 단어 뜻, 반의어, 문맥 의미 |
| 문장 삽입 미니형 | 3~4문장 짧은 지문 |
| 빈칸 추론 미니형 | 자체 생성 짧은 문단 |
| 연결어 선택 | however, therefore 등 |
| 주제문 찾기 | 짧은 문단 요약 |

영어는 **자체 생성 짧은 지문 + 검수**가 필요해.

---

## 국어

국어는 가장 조심해야 해.  
특히 문학 지문은 저작권 이슈가 크고, 독서 지문도 품질 검수가 어렵다.

초기에는 이렇게 가야 해.

| 변형 유형 | 예시 |
|---|---|
| 독서 구조 파악 | 중심문장 찾기 |
| 선지 판단 훈련 | 옳은/틀린 근거 찾기 |
| 개념어 회독 | 인과, 대조, 예시, 정의 |
| 짧은 자체 지문 | 300~500자 비문학 |
| 근거 문장 매칭 | 선지와 근거 연결 |

국어는 “긴 기출형 문제 생성”보다 **근거 찾기·선지 판단 훈련**이 먼저야.

---

## 탐구

탐구는 베리에이션 만들기 좋다.

| 변형 축 | 예시 |
|---|---|
| 개념 비교 | 롤스 vs 노직 |
| 사례 적용 | 분배 정의 사례 판단 |
| 자료 해석 | 표, 그래프, 실험 결과 |
| 선지 변형 | ㄱㄴㄷ 조합 |
| 함정 변형 | 개념 혼동 유도 |

탐구는 수학 다음으로 MVP에 적합해.

# 9. 베리에이션 생성 파이프라인

Agent 구조는 이렇게 잡으면 돼.

```text id="1ir5mg"
[문항 분석 Agent]
원문/학생 오답 문제 분석

↓
[문항 DNA 추출 Agent]
개념, 풀이전략, 조건, 오답원인 분해

↓
[베리에이션 정책 Agent]
학생 숙련도와 목적에 맞는 V레벨 선택

↓
[문항 변형 Agent]
숫자/조건/표현/요구값/형식 변형

↓
[오답 선지 Agent]
학생이 자주 틀린 원인을 반영한 선지 생성

↓
[난이도 Agent]
L1~L5 난이도 예측

↓
[정합성 평가 Agent]
정답 유일성, 풀이 검증, 수능형 품질 평가

↓
[출제 Agent]
오늘의 회독 / 오답던전 / 망각방어전 / 4점보스에 배치
```

# 11. 실제 서비스에서는 “베리에이션 묶음”을 만든다

문제 하나에서 하나만 만들지 말고, 5개짜리 묶음을 만들어야 해.

예:

| 번호 | 유형 | 목적 |
|---:|---|---|
| 1 | OX | 개념 확인 |
| 2 | 빈칸 | 공식/조건 회수 |
| 3 | 단답형 | 기본 적용 |
| 4 | 객관식 | 수능형 적응 |
| 5 | 오류 찾기 | 오답 원인 교정 |

이걸 **회독 세트**라고 부르면 좋아.

> 문제 하나를 올리면 AI가 회독 세트 5개를 만든다.

이게 순공대장에 딱 맞아.

# 12. MVP에서는 어디까지 하면 되나

처음부터 전과목 베리에이션은 무리야.  
MVP는 이렇게 가야 해.

## 1차 MVP 베리에이션 범위

| 항목 | 범위 |
|---|---|
| 과목 | 수학 우선 |
| 유형 | 수열, 함수, 확률 중 1~2개 |
| 베리에이션 | V1~V3까지만 |
| 문제 형식 | OX, 빈칸, 단답형, 객관식 |
| 검증 | Solver 2개 + 정답 일치 |
| 출제 | 오늘의 회독, 망각방어전 |

V5 4점보스는 나중에 넣어도 돼.  
처음에는 **“학생이 틀린 문제를 다른 형태로 다시 풀게 만든다”**만 증명하면 충분해.

# 13. 순공대장식 최종 정의

이렇게 정리하면 좋아.

> **순공대장의 베리에이션 Agent는 원문 문제를 복사·변형하는 엔진이 아니라, 문항의 개념·풀이전략·오답원인·조건 구조를 분해한 뒤 학생 숙련도에 맞춰 숫자, 요구값, 표현 방식, 조건 구조, 문제 형식을 단계적으로 바꾸는 AI 회독 생성 엔진이다.**

사업계획서용으로는 더 짧게:

> 학생이 틀린 문제를 AI가 문항 DNA로 분해하고, 같은 개념을 OX·빈칸·단답형·수능형·보스형으로 단계 변형하여 반복 출제한다. 이를 통해 단순 재풀이가 아니라 개념 회수, 오답 원인 교정, 실전 전이를 동시에 유도한다.

핵심 한 줄은 이거야.

> **같은 문제를 다시 푸는 게 아니라, 같은 약점을 다른 얼굴로 다시 만나게 만든다.**

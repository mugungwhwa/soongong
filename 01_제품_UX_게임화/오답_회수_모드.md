# 오답 회수 모드

> 원본 소스: `오답 회수 모드.txt`  
> 분류: **제품 UX·게임화**

---

완전 좋아. 아니, 이건 **있으면 좋은 기능**이 아니라 순공대장의 핵심 기능으로 넣어야 해.

정확히 말하면 이 기능은 단순 “유사문제 추천”이 아니라:

> **오답 풀이 → 오답 원인 진단 → 유사문항 베리에이션 → 다시 맞힐 때까지 회독**
  
이 구조야.

기존 순공대장 설계에서도 문제 하나를 그대로 변형하는 게 아니라 **문항 DNA를 추출한 뒤, 변형 축·난이도·오답 원인을 반영해 새 문제를 만드는 베리에이션 Agent**가 핵심 엔진으로 잡혀 있어. fileciteturn0file3

---

## 기능 이름은 이렇게 잡으면 좋다

### 1순위 추천

**오답회수 모드**

말이 제일 직관적이야.

> “틀린 문제를 다시 맞히는 순간까지 회수한다.”

### 서비스 안에서는 이렇게 표현 가능

```text id="4hkxc7"
오답회수 시작
비슷한 문제 3개로 다시 잡자.
```

또는 마스코트 캐릭터 ‘순공이’(가안) 말투로:

```text id="8mndiq"
“틀린 이유 알았으면, 바로 비슷한 거 하나 더 간다.”
“해설만 보고 끝내면 또 까먹는다.”
“이번엔 숫자만 바꿨다. 진짜 이해했는지 보자.”
```

---

## 핵심 플로우

```text id="cnta9d"
학생이 틀림
↓
AI 오답 원인 분석
↓
짧은 해설 제공
↓
바로 유사문항 1개 출제
↓
맞히면 XP + 오답회수
↓
또 틀리면 더 쉬운 변형으로 회귀
↓
최종적으로 맞히면 회독 성공
```

이게 진짜 듀오링고스럽다. 듀오링고도 틀린 걸 그냥 해설만 주고 끝내지 않고 다시 연습하게 만들잖아. 순공대장은 이걸 수능 오답에 맞게 만든 거야.

---

## 중요한 건 “비슷한 문제”를 6단계로 나누는 것

무작정 유사문제 많이 주면 안 돼.  
학생이 틀렸을 때는 가까운 변형부터 줘야 해.

| 단계 | 이름 | 예시 | 목적 |
|---:|---|---|---|
| V0 | 원문 회독 | 방금 틀린 문제 다시 풀기 | 기억 회수 |
| V1 | 숫자 변형 | 숫자, 계수만 변경 | 같은 풀이 재훈련 |
| V2 | 요구값 변경 | 일반항 → 특정항 | 이해 확인 |
| V3 | 표현 변경 | 식 → 표/그래프 | 유형 인식 |
| V4 | 조건 변경 | 조건 추가/역방향 질문 | 수능형 적응 |
| V5 | 개념 결합 | 수열+함수 등 | 4점보스 |

예를 들어 수학 수열 점화식에서 틀렸다면:

```text id="12v04o"
1단계: 해설 확인
2단계: 숫자만 바꾼 문제
3단계: 특정항 구하기 문제
4단계: 표로 제시된 문제
5단계: 조건이 바뀐 문제
6단계: 보스형 문제
```

기존 설계에서도 “방금 오답이면 V0/V1, 정답이지만 오래 걸리면 V2, 빠른 정답이면 V3, 숙련도 80 이상이면 V5 보스전”처럼 학생 상태에 따라 변형 강도를 조절하는 방식이 맞다고 정리돼 있어. fileciteturn0file3

---

## 오답 원인별로 다른 유사문제를 줘야 함

여기가 진짜 차별점이야.

예를 들어 같은 수학 문제를 틀렸어도 이유가 다르면 다음 문제가 달라야 해.

| 오답 원인 | 다음 출제 |
|---|---|
| 개념 미흡 | OX / 빈칸 / 개념 카드 |
| 계산 실수 | 숫자만 바꾼 짧은 재계산 |
| 조건 누락 | “반드시 써야 하는 조건 고르기” |
| 인덱스 혼동 | n/n+1 비교 문제 |
| 선지 착각 | 오류 찾기 / 선지 판단 문제 |
| 시간 부족 | 풀이 순서 배열 / 단계 분해 |

그러니까 순공대장은 이렇게 말할 수 있어.

> **“정답만 알려주는 AI가 아니라, 틀린 이유에 맞는 다음 문제를 바로 내는 AI”**

이건 꽤 강한 차별점이야.

## UX는 이렇게 만들면 좋다

학생이 틀렸을 때:

```text id="pwe9nx"
틀렸어.

오답 원인:
n과 n+1 인덱스 혼동

짧은 해설:
aₙ₊₁을 구할 때 n에 1을 넣으면 a₂가 됩니다.

[비슷한 문제로 바로 회수하기]
```

버튼 누르면:

```text id="97r3ba"
오답회수 1/3

이번엔 숫자만 바꿨어.
진짜 이해했는지 확인해보자.
```

맞히면:

```text id="f2j2nw"
오답회수 성공!

+30XP
수열 점화식 숙련도 +3
망각위험 감소
다음 회독: 3일 후
```

또 틀리면:

```text id="3ic84p"
아직 흔들린다.
이번엔 더 짧게 간다.

OX:
점화식에서 aₙ₊₁은 다음 항을 의미한다. O/X
```

## 단, “계속 무한 출제”는 조심해야 함

학생이 계속 틀리면 무한정 유사문제를 던지면 안 돼. 피로감이 생겨.

추천은 이거야.

| 상황 | 처리 |
|---|---|
| 1회 오답 | 해설 + V1 유사문항 |
| 2회 오답 | L1 개념 카드 / OX로 회귀 |
| 3회 오답 | 오늘은 중단, 내일 재회독 예약 |
| 정답 성공 | 오답회수 완료, 3일 후 복습 |
| 빠른 정답 | 다음엔 V2/V3로 강화 |

순공대장 난이도 설계에서도 한 문제 반응만 보고 바로 흔들기보다 **최근 5개 같은 개념/유형 문제 기준으로 정답률, 풀이시간, 힌트 사용률, 자신감 등을 보고 난이도를 조정**하는 구조가 적합하다고 잡혀 있어. fileciteturn0file2

## 제품적으로는 이 기능이 엄청 중요함

왜냐하면 수능생이 진짜 원하는 건 “해설”이 아니야.

해설은 이미 인강에도 있고, 문제집에도 있어.  
근데 학생은 해설을 보고도 다시 틀려.

순공대장이 줄 수 있는 가치는 이거야.

> **“해설을 이해한 것처럼 느끼는 상태”에서 끝내지 않고,  
> 바로 비슷한 문제를 풀게 해서 진짜 이해했는지 검증한다.**

이게 순공대장의 핵심 문장과도 맞아.

> **순공대장은 문제를 풀어주는 AI가 아니라, 다시 풀게 만드는 AI다.**

## MVP에는 이렇게 넣자

처음부터 전 과목 유사문항 생성은 무리야.  
MVP에서는 이렇게 가면 좋아.

### MVP 1단계

```text id="4qgbyh"
오답 후
→ 오답 원인 선택
→ 개념 카드 / OX / 빈칸 생성
→ 다음 회독 예약
```

### MVP 1.5단계

```text id="ru4yp1"
수학 중심
→ 숫자/조건 변형 유사문항 생성
→ Solver Agent로 정답 검증
→ 오답회수 모드 제공
```

### MVP 2단계

```text id="onvkrn"
탐구/영어까지 확대
→ 유형 카드 RAG 기반 생성
→ 오답던전/망각방어전과 연결
```

수학은 템플릿 + 변수 치환 + 자동 검증이 가능해서 먼저 넣기 좋고, 국어·영어 긴 지문형은 저작권과 품질 검수 때문에 뒤로 미루는 게 안전해. fileciteturn0file0

## 결론

이 기능은 무조건 넣어야 해.

다만 표현은 **“AI 유사문제 무한 생성”**이 아니라:

> **오답회수 기능**  
> 틀린 이유에 맞춰 비슷한 문제를 단계적으로 다시 풀게 하는 AI 회독 루프

로 잡는 게 좋아.

순공대장 핵심 루프는 결국 이렇게 돼.

```text id="lju03a"
틀림
→ 왜 틀렸는지 안다
→ 비슷한 문제를 다시 푼다
→ 맞힌다
→ 3일 뒤 또 회독한다
→ 까먹기 전에 다시 회수한다
```

이게 진짜 수능판 듀오링고 기능이야.

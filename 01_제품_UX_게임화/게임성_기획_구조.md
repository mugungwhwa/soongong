# 게임성 기획 구조

> 원본 소스: `게임성 기획 구조.txt`  
> 분류: **제품 UX·게임화**

---

좋아. **순공대장 게임성 기획 v0.1**은 이렇게 잡으면 돼.

핵심은 이거야.

> **겉은 학습 앱, 속은 회독 게임.**  
> 학생이 “공부했다”에서 끝나지 않고, **오늘의 회독을 깨고 다시 오게 만드는 시스템**.

기존 UI 방향도 “게임처럼 보이지 않지만 매일 회독을 깨는 느낌이 있는 학습 앱”으로 잡혀 있고, 스트릭·XP·회독 완료·경로 노드를 핵심 게임 장치로 두는 방향이 맞아. fileciteturn2file1

# 1. 전체 게임성 구조

순공대장은 게임성을 5개 층으로 나누면 된다.

| 층 | 기능 | 목적 |
|---|---|---|
| 1층 | **달성** | 오늘 회독을 끝내게 함 |
| 2층 | **XP / 기억 HP** | 즉시 보상 |
| 3층 | **뱃지 / 트로피** | 특정 행동을 기념 |
| 4층 | **등급 / 레벨** | 장기 성장감 |
| 5층 | **순공리그 / 푸시** | 재방문과 경쟁 |

가장 중요한 건 **등급, 뱃지, 리그를 같은 보상으로 섞지 않는 것**이야.

- **달성** = 오늘 해야 할 것
- **뱃지** = “이 행동을 해냈다”는 기록
- **등급** = 누적 성장
- **리그** = 이번 주 경쟁
- **푸시** = 다시 들어오게 만드는 트리거

# 2. 핵심 루프

```text id="5y3623"
문제사진 / 인강 기록 / 캡처
↓
AI 분석
↓
오늘의 회독 3개 생성
↓
회독 플레이
↓
XP + 기억 HP + 뱃지 진행도
↓
등급 상승 / 리그 점수 반영
↓
다음 회독 예약
↓
푸시로 재방문
```

순공대장 설계에서도 듀오링고식 기능을 오늘의 회독 경로, 회독 XP, 순공리그, 기억 HP, 오답던전, 까먹기 전 알림으로 변환하는 구조가 제안되어 있어. fileciteturn2file0

# 3. 달성 시스템

## 3-1. 오늘의 달성

매일 앱을 켜면 학생에게 보이는 목표는 딱 3개면 좋다.

```text id="6x9ivn"
오늘의 회독 3개

1. 수학 · 점화식 · 7일 전 오답
2. 영어 · 빈칸추론 · 인강 메모 기반
3. 생윤 · 롤스/노직 · 개념 혼동
```

목표는 “수학 2시간 공부해라”가 아니라:

> **7일 전 틀린 점화식 하나 다시 풀어라.**

이렇게 작아야 한다.

## 3-2. 달성 종류

| 달성 | 조건 | 보상 |
|---|---|---|
| 오늘의 회독 완료 | 필수 회독 3개 완료 | XP + 스트릭 유지 |
| 망각방어 성공 | 예정된 복습 문제 정답 | 기억 HP 회복 |
| 오답회수 | 예전에 틀린 문제를 다시 맞힘 | 오답회수 점수 |
| 순공 인정 | 15~60분 집중 기록 | 순공 XP |
| 회독 퍼펙트 | 오늘 회독 전부 힌트 없이 정답 | 보너스 XP |
| 빠른 회수 | 3분 이내 짧은 회독 완료 | 미니 보상 |

홈 화면도 문제 리스트가 아니라 “오늘 다시 봐야 하는 이유”가 보이는 퀘스트 카드로 설계하는 게 맞고, 퀘스트 카드에는 과목/단원, 망각위험, 예상 시간, 보상 XP, CTA가 들어가야 한다. fileciteturn2file1

# 4. XP / 기억 HP

## 4-1. XP

XP는 성적 보상이 아니라 **회독 행동 보상**으로 써야 해.

| 행동 | XP |
|---|---:|
| 오늘의 회독 1개 완료 | +20XP |
| 오답 재풀이 성공 | +30XP |
| 망각방어 성공 | +40XP |
| 힌트 없이 정답 | +10XP |
| 7일 전 문제 재정답 | +20XP |
| 14일 망각방어 성공 | +50XP |
| 4점보스 클리어 | +80XP |

## 4-2. 기억 HP

하트 대신 **기억 HP**를 쓰면 순공대장스럽다.

```text id="ruwofr"
기억 HP 3/5
```

| 상황 | 변화 |
|---|---|
| 망각방어 성공 | HP +1 |
| 오늘 회독 미완료 | HP -1 |
| 같은 오답 반복 | HP -1 |
| 3일 연속 회독 완료 | HP 최대치 회복 |
| 7일 스트릭 달성 | HP 보호막 1개 |

기억 HP는 학생을 벌주는 장치가 아니라, “까먹기 직전”을 시각화하는 장치야. 기존 설계에서도 기억 HP와 망각위험 배지를 홈·오답던전·내 기록에 반복 노출하는 방향이 제안되어 있어. fileciteturn1file10

# 5. 뱃지 / 트로피

여기서 구분이 중요해.

> **뱃지 = 많이 모으는 작은 업적**  
> **트로피 = 시즌/대형 업적**

## 5-1. 뱃지 종류

| 카테고리 | 뱃지 예시 | 획득 조건 |
|---|---|---|
| 시작 | 첫 회독 뱃지 | 첫 오늘의 회독 완료 |
| 스트릭 | 7일 불꽃 | 7일 연속 회독 |
| 회독 | 3회독러 | 같은 개념 3회 복습 |
| 망각방어 | 기억수비수 | 7일 전 문제 정답 |
| 오답회수 | 오답회수꾼 | 오답 10개 다시 맞힘 |
| 과목 | 수열 사냥꾼 | 수열 회독 20개 완료 |
| 인강 | 인강회수러 | 인강 메모 기반 회독 10개 |
| 집중 | 60분 순공러 | 인정 순공 60분 달성 |
| 보스 | 4점보스 입문 | 4점보스 첫 클리어 |

## 5-2. 트로피 종류

| 트로피 | 조건 | 의미 |
|---|---|---|
| 주간 회독왕 | 주간 회독 30개 이상 | 꾸준함 |
| 망각방어 챔피언 | 주간 망각방어 성공률 80% 이상 | 기억 유지 |
| 오답회수 마스터 | 주간 오답회수 20개 이상 | 약점 극복 |
| 순공리그 TOP10 | 리그 상위 10위 | 경쟁 성과 |
| 월간 순공대장 | 월간 회독 완료율 90% 이상 | 장기 성취 |

## 5-3. 뱃지 희귀도

| 등급 | 색 | 예시 |
|---|---|---|
| 일반 | 그린 | 첫 회독 |
| 희귀 | 블루 | 7일 스트릭 |
| 영웅 | 퍼플 | 14일 망각방어 성공 |
| 전설 | 골드 | 월간 회독 완료율 90% |

뱃지는 앱이 너무 게임처럼 보이지 않도록 **내정보 / 기록 화면 안의 컬렉션**으로 두는 게 좋다. 홈에는 “새 뱃지 획득!” 정도만 짧게 보여주면 돼.

# 6. 등급 / 레벨

등급은 두 개로 나누자.

## 6-1. 사용자 전체 등급

이건 누적 XP 기반.

| 등급 | 누적 XP | 느낌 |
|---|---:|---|
| 순공입문 | 0 | 시작 |
| 순공러 | 500 | 습관 형성 |
| 순공대장 | 1,500 | 꾸준한 유저 |
| 순공도사 | 3,500 | 회독 고수 |
| 순공마왕 | 7,000 | 장기 유저 |
| 순공전설 | 12,000 | 최상위 |

## 6-2. 개념 숙련 등급

이건 과목/단원별로 따로 둔다.

```text id="10jfva"
수학 · 수열 점화식
숙련도 68점
등급: 안정권 직전
```

| 숙련도 | 상태 | UI |
|---:|---|---|
| 0~39 | 위험 | 빨강 |
| 40~59 | 불안정 | 노랑 |
| 60~79 | 안정권 | 그린 |
| 80~100 | 장기기억 후보 | 딥그린 / 골드 |

이렇게 해야 학생이 “내 전체 레벨은 올랐는데, 수열은 아직 위험하네”를 이해할 수 있어.

# 7. 순공리그

순공리그는 MVP 1차보다는 **MVP 1.5차**가 좋아. 처음부터 리그를 넣으면 콘텐츠보다 경쟁만 앞설 수 있어.

## 7-1. 리그 구조

| 리그 | 인원 | 승급 |
|---|---:|---:|
| 브론즈 | 50명 | 상위 10명 |
| 실버 | 50명 | 상위 10명 |
| 골드 | 50명 | 상위 10명 |
| 플래티넘 | 50명 | 상위 8명 |
| 다이아 | 50명 | 상위 5명 |
| 순공마스터 | 50명 | 상위 3명 |

## 7-2. 리그 점수 공식

단순 순공시간만으로 경쟁하면 타이머 뻥튀기가 생긴다. 그래서 점수는 행동 기반으로 가야 해.

```text id="3digff"
주간 리그 점수 =
인정 순공분 × 1점
+ 회독 완료 × 10점
+ 오답회수 × 20점
+ 망각방어 성공 × 15점
+ 힌트 없이 정답 × 5점
+ 4점보스 클리어 × 30점
```

단, **인정 순공시간은 하루 60분까지만 리그 점수 반영**. 기존 설계에서도 순공리그는 인정 순공시간, 회독 완료 수, 오답회수 수, 망각방어 성공률을 기준으로 하고, 하루 1시간 cap을 둬 과몰입과 밤샘 보상을 막는 방향이 적절하다고 정리되어 있어. fileciteturn2file0

## 7-3. 리그 화면

```text id="b8fgyj"
이번 주 실버 III

12위 / 50명
승급까지 38점

내 점수
- 회독 완료 18개
- 오답회수 7개
- 망각방어 성공률 72%
- 인정 순공 280분

[오늘 회독하고 +40점 받기]
```

중요한 건 리그 화면에서도 CTA는 “공부 더 해라”가 아니라:

> **오늘 회독하고 점수 받기**

여야 해.

# 8. 푸시 설계

푸시는 “잔소리”가 아니라 **망각위험 알림**이어야 한다.

## 8-1. 푸시 종류

| 종류 | 조건 | 예시 |
|---|---|---|
| 아침 계획 | 오늘 회독 생성 | “오늘 회독 3개 열렸어. 수학 1개부터 가자.” |
| 망각위험 | 7일/14일 복습 도래 | “7일 전 틀린 점화식, 오늘 안 보면 위험해.” |
| 스트릭 위기 | 오늘 미완료 + 저녁 | “오늘 회독 1개만 하면 7일 스트릭 유지.” |
| 리그 위기 | 승급/강등 근처 | “실버 유지까지 20점. 회독 1개면 충분해.” |
| 뱃지 직전 | 조건 80% 달성 | “오답회수 뱃지까지 2개 남았어.” |
| 복귀 | 2~3일 미접속 | “쌓인 망각위험 4개. 짧게 하나만 회수하자.” |

기존 설계에서도 푸시/위젯은 “까먹기 전 알림”으로 설계하고, 예시는 “오늘 수학 점화식 1개만 회독하면 스트릭 유지”, “7일 전 틀린 문제, 오늘 망각위험 높음”처럼 잡혀 있어. fileciteturn2file0

## 8-2. 푸시 시간대

| 시간 | 목적 | 문구 |
|---|---|---|
| 오전 8시 | 오늘 목표 인지 | “오늘 회독 3개 열렸어.” |
| 오후 6시 | 방과/학원 전후 리마인드 | “수학 점화식 1개만 먼저 회수하자.” |
| 밤 10시 | 스트릭 방어 | “오늘 회독 1개 남았다. 3분이면 끝.” |

## 8-3. 푸시 제한

수험생 앱이라 푸시는 조심해야 해.

```text id="pqclsx"
하루 최대 2회
밤 11시 이후 푸시 금지
자극적 문구 금지
성적 보장 문구 금지
랭킹 압박 과도하게 금지
```

피해야 할 문구:

```text id="fbom30"
너 이러면 망한다
1등급 못 간다
공부 안 하면 끝이다
```

좋은 문구:

```text id="piye67"
까먹기 전에 하나만 회수하자.
오늘 회독 1개만 하면 스트릭 유지.
7일 전 오답, 지금 풀면 기억 HP 회복.
```

교육 서비스라 성적 보장, 적중, 합격 보장 문구는 피해야 한다는 방향도 기존 법무/카피 가이드와 맞아. fileciteturn1file5

# 9. 화면별 게임성 강도

순공대장은 처음부터 전체를 게임처럼 만들면 안 된다. 기존 UI 가이드에서도 홈은 학습 앱처럼, 결과와 서브 모드에서만 게임성을 올리는 방향이 적합하다고 정리되어 있어. fileciteturn2file1

| 화면 | 게임성 강도 | 넣을 요소 |
|---|---:|---|
| 홈 | 30% | 스트릭, 레벨, 회독 경로 |
| 회독 플레이 | 20% | 진행도, 기억 HP |
| 결과 화면 | 50% | XP, 뱃지 진행도, 마스코트 반응 |
| 오답던전 | 60% | 약점 공략, 던전 느낌 |
| 망각방어전 | 60% | 위험도, 기억 HP |
| 순공리그 | 70% | 순위, 승급, 주간 보상 |
| 4점보스 | 80% | 선택형 도전 |

# 11. 최종 한 줄

> **순공대장 게임성은 “오래 공부한 사람”이 아니라 “까먹기 전에 다시 푼 사람”을 보상해야 한다.**

그래서 리그도, 뱃지도, 푸시도 전부 이 기준으로 잡으면 돼.

```text id="vi7bld"
오늘의 회독 완료
오답회수
망각방어 성공
회독 스트릭
인정 순공 60분 cap
```

이 5개가 순공대장 게임성의 핵심 지표야.

# 순공대장 UI 설계

> **1차 완성안 시안: `app_UI.png`, `web_ui.png` — 이미지가 SSoT.** 본 문서의 텍스트와 시안이 충돌하면 **이미지가 우선**한다.
> **마스코트 캐릭터: 작업명 "순공이", 형태 "듀공(dugong)" — 모두 가안.** 추후 별도 일러스트 작업으로 확정.
> **용어**: 본 문서에서 "회독"은 학습 메커니즘 명칭, "회독퀘스트"는 사용자에게 노출되는 UI 라벨이다. 같은 개념을 가리킨다.
> 분류: **제품 UX·게임화**

---

순공대장 디자인은 **밝고 친근한 매일 회독퀘스트 정원**으로 간다.

> **듀오링고의 리텐션 구조 + 한국 학습앱(클래스101/콴다)의 학습 카드 메타 + 카카오 헤이바이브의 친근 마스코트 톤 + 연두빛 학습 정원**

(상세 비교 reference는 §1-2 "비교 레퍼런스" 참조)

기존 v1.0에서 잡았던 "Dark Study RPG / 다크 네이비 / 회독마왕" 방향은 폐기한다. 수험생에게 너무 무겁고, 매일 들어오게 만드는 데 압박감이 도리어 방해가 된다는 판단. 최신 시안(`app_UI.png`, `web_ui.png`)에서 잡힌 라이트 톤이 정답.

# 1. 디자인 콘셉트

## 이름

**순공대장**

## 세계관 톤

“공부 앱”이 아니라
**마스코트 캐릭터 '순공이'(가안)가 매일 회독퀘스트를 함께 걸어주는 공부 친구**

## 한 줄 디자인 키워드

> **오늘 회독퀘스트 3개, 까먹기 전에 가볍게**

## 비교 레퍼런스 (designer agent 가이드)

본 시안의 톤은 단일 reference 차용이 아니라 **4개 영역에서 부분 차용**한 결과다. designer agent가 화면을 그릴 때 각 영역의 정확한 reference를 참고할 것:

| 영역 | 정확한 reference | 차용된 요소 |
|---|---|---|
| 매일 게임화 루프 | **듀오링고 (Duolingo)** | 스트릭, XP, HP, 오늘의 3개, 마스코트, 그린 톤 |
| 친근 마스코트 + 라이트 그린 | **카카오 헤이바이브 / 라인 캐주얼 라인** | 둥근 캐릭터, 친근 카피, 게임센터 친근감 |
| 학습 카드 메타 정보 | **클래스101 / 산타토익 / 콴다** | 위험도 배지, 난이도 칩, 예상 시간, 보상 XP |
| 한국 SaaS 디자인 표준 (코드 수준) | **Pretendard + shadcn/ui** | 둥근 카드(20px), 부드러운 그림자, 큰 타이포 위계 |

### 의도적으로 차용하지 않는 reference

| reference | 차용 안 하는 이유 |
|---|---|
| **토스** | 시그니처 컬러(파랑), 미니멀(한 카드 한 액션), 마스코트 없음 — 본 시안과 본질적으로 다름. (단 Pretendard 폰트는 토스가 만들어 오픈소스화한 것이라 자연스럽게 사용) |
| **듀오링고의 세로 snake path** | 학습 트리 시각화는 P5 sub-plan에서 spike 후 결정. 기본은 카드 리스트 |
| **다크 RPG / 다크 판타지 UI** | v1.0에서 폐기됨. 압박감으로 매일 들어오기를 방해 |

> **note**: v1.0 (`UI 설계.md` 초안)에서 "토스/스픽의 깔끔함"이라는 표현이 있었으나, 이는 실제로 "한국 SaaS 디자인 표준 (Pretendard + 둥근 + 그림자 + 위계)"를 가리킨 short-hand였음. v2.3에서 명시적으로 분리.

# 6. 마스코트 캐릭터 (가안 — 추후 확정)

> ⚠️ **본 섹션은 가안이다.** 캐릭터 이름·외형·말투는 별도 일러스트 작업 단계에서 확정한다.
> 시안 이미지에 표시된 마스코트는 작업용 임시안이며, 최종 디자인은 변경될 수 있다.

## 작업명 (가안)

- **이름**: 순공이 (가안)
- **형태**: 듀공(dugong) 모티프 (가안)
- **컬러**: 메인 그린 톤, 시안 이미지 색조와 통일

## 왜 듀공인가 (가안 근거)

듀공은 둥글고 친근한 해양 포유류로, 표정이 부드럽고 비공격적이라 학생을 응원하는 공부 친구 이미지에 잘 맞는다. 매너티·해마와 비교해 인지도는 낮지만 그만큼 **한국 캐릭터 IP로 자리잡기 좋은 여백**이 있다. 또한 "해마(hippocampus, 기억)" 메타포 대신 듀공을 쓰면, 친근감을 우선하면서 추후 "기억" 컨셉은 캐릭터 소품(책·별 등)으로 보완할 수 있다.

> 최종 결정 전이며, 일러스트 외주 단계에서 다른 모티프(해마/매너티/물범 등)로 교체 가능.

## 캐릭터 방향 (가안)

| 요소 | 가안 |
|---|---|
| 형태 | 둥근 듀공 — 머리 60% / 몸 40%, 짧은 지느러미 |
| 표정 | 친근 90%, 응원 10% |
| 톤 | 매일 응원해주는 공부 친구 |
| 컬러 | 메인 그린 톤, 배 부분 옅은 민트 `#DFF5DC` |
| 소품 | 책, 별, 작은 왕관 (선택) |
| 비율 | 작고 둥글게, 압박감 0 |

## 순공이 말투 가이드 (친근/응원형)

```text
"오늘 회독퀘스트 3개 열렸어. 같이 가볼까?"
"7일 전에 틀린 점화식, 한 번만 다시 보자."
"회독퀘스트 성공! 기억 HP 회복됐어."
"오늘은 보스 말고 가볍게 L2부터 가자."
"잠깐, 이거 며칠 전에 헷갈렸지? 가볍게 회수해보자."
```

## 절대 쓰지 않을 말투 (구 회독마왕 톤, 폐기)

```text
❌ "너 이거 전에 틀렸지?"
❌ "까먹기 직전이다."
❌ "오늘 안 풀면 기억 HP 깎인다."
```

# 7. 핵심 컴포넌트 디자인

## 1) 퀘스트 카드 (라이트 버전)

```text
┌───────────────────────────────┐
│ ① 수학 · 수열 점화식           │
│                               │
│  🔴 망각위험 높음   L3   4분   │
│                               │
│  7일 전 오답 · 인덱스 혼동      │
│                               │
│  보상: +40 XP   망각방어 +1    │
│                               │
│  [ 회독퀘스트 시작 ]            │
└───────────────────────────────┘
```

스타일:
| 요소 | 값 |
|---|---|
| 카드 배경 | `#FFFFFF` |
| 카드 라운드 | 20px |
| 그림자 | `0 2px 8px rgba(0,0,0,0.06)` |
| 번호 동그라미 | 민트 fill, 흰 숫자, 32x32 |
| 배지 | radius 999px (pill), 텍스트 12px, 패딩 4/10 |
| 난이도 칩 | 흰 fill + 옅은 그린 테두리 |
| CTA | 메인 그린 fill, radius 16, 높이 48 |

## 2) 기억 HP

```text
❤️ 5/5
```

- 하트는 메인 그린 톤 (붉은 하트 아님 — 위협감 줄임)
- "5/5" 숫자 강조, "기억 HP" 텍스트는 보조
- 풀로 차면 라임 fill, 닳을수록 회색

## 3) 망각위험 배지

```text
🔴 망각위험 높음     bg #FCE8E8 / text #E85C5C
🟠 오늘 회독퀘스트 권장   bg #FFF1DF / text #C77A3A
🔵 안정권            bg #E6F0FC / text #4A7FBF
```

- pill 형태 (radius 999)
- 작은 원형 점 + 텍스트

# 14. 다크모드 정책

**MVP는 라이트 단일.** 다크모드는 출시 후 8주 이후 검토.

이유:
- 이미지 시안이 라이트 단일로 잡혀있음
- 다크모드를 같이 만들면 토큰 / 일러스트 / 컴포넌트 작업이 2배
- 학생 대상 앱이라 야간 학습 니즈는 있지만, OS 다크모드를 강제 follow하지 않는 게 안전

# 15. 접근성 (a11y) 가이드

- 모든 텍스트는 배경 대비 WCAG AA(4.5:1) 이상
- 민트 그린 단독으로 정보 전달 금지 — 항상 텍스트/아이콘 동반
- 위험도 배지는 색만이 아니라 텍스트("높음/권장/안정")로도 구분
- 버튼 최소 hit area 44x44 (모바일)
- 키보드 포커스 outline 절대 제거 금지

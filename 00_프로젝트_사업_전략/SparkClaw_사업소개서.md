# SparkClaw용 순공대장 사업소개서 방향

## 1. 한 줄 정의

**순공대장은 수능생을 위한 듀오링고형 AI 회독 리텐션 엔진입니다.** 문제사진·인강기록·오답을 AI가 매일 회독 퀘스트로 바꿉니다.

더 강한 버전 (Deck용):

> **콴다는 학생의 '막힘'을 풀고, 순공대장은 학생의 '까먹음'을 풉니다.**
>
> 콴다는 1세대 (질문 → 답), 순공대장은 2세대 (풀이 → 기억). 시장이 1세대를 흡수했으니 2세대로 갈 때.

학술 근거: Karpicke & Roediger (2008, *Science*) — 단순 학습 그룹은 1주 뒤 80%를 까먹었고, 인출 연습 그룹은 80%를 기억. **같은 시간, 4배 차이.** 한국 에듀테크가 본격적으로 활용 안 한 영역.

SparkClaw는 "AI가 당신의 팀입니다"라는 메시지로 1인·소규모 AI 창업자를 대상으로 하고, AI를 단순 도구가 아니라 팀원처럼 활용하는 창업가를 발굴한다고 설명하고 있어. 그래서 순공대장은 **AI Agent가 제품의 기능이자, 1인 창업자가 실행하는 방식**이라는 점을 전면에 세우는 게 맞아. ([sparkclaw.co.kr](https://www.sparkclaw.co.kr/))

## 2. 문제 정의

한국 수능생은 이미 많은 시간을 공부에 쓴다. 문제는 **공부를 안 하는 것**이 아니라, 공부한 것이 장기기억으로 남지 않는다는 점이다.

학생은 인강을 보고, 문제를 풀고, 오답을 표시하지만 대부분 여기서 멈춘다. 언제 다시 풀어야 하는지 모르고, 오답을 체계적으로 회수하지 못하고, 공부시간은 쌓이지만 기억은 빠르게 사라진다. 그래서 순공대장이 해결하는 문제는 “강의 부족”이 아니라 **복습 실행 부족**이다.

시장도 충분히 크다. 2025년 초·중·고 사교육비 총액은 27.5조 원이고, 고등학생 사교육비만 7.8조 원 규모다. 전체 사교육 참여율은 75.7%, 고등학생 참여율은 63.0%로 집계됐다. ([mods.go.kr](https://mods.go.kr/boardDownload.es?bid=11758&list_no=444220&seq=1)) 특히 2025년 고등학생 사교육 참여 학생의 월평균 지출은 79.3만 원으로, 참여 학생 기준 지출 강도는 여전히 높다. ([mods.go.kr](https://mods.go.kr/boardDownload.es?bid=11758&list_no=444220&seq=1))

## 3. 솔루션

순공대장은 학생이 공부한 흔적을 AI가 회독 가능한 학습 객체로 바꾼다.

```text
문제사진 / 인강기록 / 캡처 / 메모
→ AI 분석
→ 과목·단원·유형·오답원인 태깅
→ 1일/3일/7일/14일 복습 예약
→ 오늘의 회독 생성
→ 오답던전 / 망각방어전 / 순공리그 반영
```

앱의 첫 화면은 대시보드가 아니라 **오늘의 회독길**이다. 마스코트 캐릭터 ‘순공이’(가안)가 “오늘 회독 3개만 하면 기억 지킨다”처럼 짧게 안내하고, 학생은 세로 회독 경로를 따라 문제사진에서 생성된 복습 퀘스트를 클리어한다. 내부 UI 방향도 “순공이가 이끄는 듀오링고형 수능 회독 경로 앱”, 즉 **겉은 공부 앱, 속은 회독 게임**으로 잡혀 있다. fileciteturn4file1

## 4. 핵심 차별점

| 기존 서비스 | 핵심 기능 | 한계 | 순공대장 차별점 |
|---|---|---|---|
| 인강 플랫폼 | 강의 제공 | 보고 끝남 | 본 강의를 회독 퀘스트로 전환 |
| 문제은행 앱 | 문제 제공 | 개인 오답 맥락 부족 | 학생이 실제 틀린 문제 기반 회독 |
| 공부 타이머 | 시간 기록 | 기억 여부를 모름 | 순공시간 + 회독시간 + 오답회수 관리 |
| AI 풀이 앱 | 정답/해설 제공 | 학생이 다시 풀지 않음 | 오답 원인·망각위험에 따라 재출제 |
| 플래너 앱 | 일정 관리 | 문제 단위 개인화 약함 | 문제사진·인강 캡처를 자동 복습 일정화 |

핵심 메시지는 이거야.

> **수능생에게 필요한 것은 더 많은 콘텐츠가 아니라, 이미 공부한 것을 다시 꺼내게 만드는 회독 루프다.**

## 5. 기술 구조

순공대장은 단순 GPT 챗봇이 아니라 **AI Agent 기반 학습 리텐션 엔진**으로 소개해야 해.

현재 기술 백서 구조는 16개 Agent로 잡혀 있다. Input Router, OCR & Capture Parsing, Lecture Capture, Question Analysis, Question DNA, Type RAG, Variation Policy, Difficulty, Question Generation, Solver, Evaluation, Review Scheduling, Quest, Game, Compliance & Admin Review까지 이어지는 구조다. 또한 RAG는 Curriculum RAG, Type Pattern RAG, Student Memory RAG의 3층으로 설계되어 있다. fileciteturn4file0

기술 해자는 크게 3개다.

첫째, **Source-to-Quest Engine**이다. 문제사진·인강기록·캡처·메모를 단순 저장하는 것이 아니라, 과목·단원·유형·오답원인·난이도·복습시점으로 구조화한다.

둘째, **Student Memory RAG**다. 학생별로 어떤 개념을 언제 틀렸고, 어떤 오답 원인이 반복되며, 최근 5문항 정답률과 힌트 사용률이 어떤지를 누적한다.

셋째, **회독 게임화 시스템**이다. XP, 기억 HP, 스트릭, 뱃지, 등급, 순공리그를 통해 “공부했다”에서 끝나지 않고 매일 다시 들어오게 만든다. 내부 게임성 설계도 달성, XP/기억 HP, 뱃지/트로피, 등급/레벨, 순공리그/푸시의 5층 구조로 정리되어 있다. fileciteturn3file3

## 6. 시장성

1차 시장은 **수능 준비생, N수생, 독학재수생, 인강 중심 학습자**다. 2026학년도 수능 응시원서 접수자는 55만 4,174명으로 전년 대비 3만 1,504명 증가했다. ([moe.go.kr](https://www.moe.go.kr/boardCnts/viewRenew.do?boardID=294&boardSeq=104055&lev=0&m=020402&opType=N&page=1&s=moe&searchType=null&statusYN=W)) 이 중 인강을 보며 혼자 공부하는 학생, 오답관리가 약한 학생, 독학재수학원·관리형 스터디카페 이용자가 초기 타깃이다.

시장 진입 논리는 이렇게 잡으면 좋다.

| 구분 | 기준 |
|---|---|
| TAM | 국내 초·중·고 사교육비 27.5조 원 |
| SAM | 고등학생 사교육비 7.8조 원 + 수능/N수 학습시장 |
| 초기 SOM | 수능 응시자 55만 명 중 인강·오답관리 니즈가 강한 1~5% |
| 초기 결제 모델 | 월 9,900~19,900원 B2C 구독 |
| 확장 모델 | 독학재수학원/스터디센터 B2B2C, 출판사·강사 제휴, 학부모 리포트 |

보수적으로 수능 응시자 55만 명 중 2%인 약 1.1만 명이 월 9,900원으로 결제해도 연 매출 약 13억 원 수준이고, 10%인 약 5.5만 명까지 확장되면 월 9,900원 기준 연 매출 약 65억 원 규모가 된다. 이 숫자는 "시장 전체를 먹는다"가 아니라 **니치하게 시작해도 유료 구독 사업이 가능한지**를 보여주는 용도로 쓰면 좋아.

### 3~5년차 매출 시뮬레이션 (듀오링고형 멀티플 적용)

수능 + 공시생 합산 100만 사용자 시점:

| 티어 | 사용자 | 월 매출 |
|---|---|---|
| Free (광고) | 91만 | — |
| Super 9,900 | 7만 | 6.9억 |
| Max 19,900 | 1.5만 | 3.0억 |
| 가족 플랜 16,500 | 5,000 가구 | 0.8억 |
| B2B2C 학원 50곳 | — | 1.0억 |
| **합계 (월)** | | **11.7억** |

+ 시즌 IAP 연 20억 → **연 매출 약 160억**. EBITDA 29.5% (듀오링고 FY2025) 적용 시 영업이익 47억. **시리즈 B 가능 수치.**

근거: 한국 수능 응시자 55만 + 공시 20만 + 자격증 일부 = TAM 150만+. 도달 가능.

## 7. 비즈니스 모델 — 듀오링고형 하이브리드 (2026-05-19 개정)

**IAP 단독 모델은 폐기.** 듀오링고 FY2025 데이터(매출 $1.03B, 구독 83%, 침투율 9%, ARPU $6)를 학습한 결과, **구독(메인) + 시즌 IAP(보조) + B2B2C(안정)** 3중 하이브리드가 valuation/안정성/확장성에서 모두 우위.

핵심 이유:
- **Valuation 멀티플**: 구독 SaaS 매출 6~10배 vs IAP 2~4배
- **MRR 예측 가능**: 시즌 변동성 회피
- **AI 차별화 가격 인상 명분**: Max 티어가 가능
- **글로벌 확장 표준**: 일본/미국 진출 시 익숙한 모델

### 7.1 5티어 구조

| 티어 | 가격 | 핵심 가치 | 타겟 |
|---|---|---|---|
| **Free** | 0원 (광고) | 일일 회독 3개, 망각방어전 주 1회, 오답 50개 한도 | 신규 유입 깔때기 |
| **Super 순공** | 월 9,900 / 연 79,000 | 무제한 회독, 오답 무제한, 광고 제거 | 메인 유료층 |
| **순공 Max** | 월 19,900 / 연 159,000 | + AI 멘토, 무한 변형 문제, 부모 리포트 주간 | 진지한 입시생 |
| **가족 플랜** | 연 199,000 (형제 3명까지) | 가구 단위 결제 (lock-in) | 형제자매 입시 가구 |
| **B2B2C** | 학원당 별도 | 학원 대시보드 + 학부모 리포트 | 학원/관리형 |

### 7.2 Max 티어 AI 차별화 (가격 인상 명분)

Claude API 활용 — 듀오링고 Max의 천재성을 한국 수능에 이식:

- **AI 회독 코치**: 매주 학생 약점 분석 → "이번 주 가장 위험한 단원은 수열입니다" 자동 코스 추천
- **AI 풀이 분석**: 풀이 캔버스 → 사고 흐름 분석 → "이 단계에서 막혔어요. 비슷한 함정 5개 더" 자동 생성
- **AI 부모 코치**: 학부모 카톡 주간 리포트 + "이번 주 이런 격려 한마디" 제안
- **AI 모의고사 분석**: 단원별/유형별/시간배분 진단

이 기능들이 Max 가격 정당화. Claude API 비용은 매출 따라 자동 스케일.

### 7.3 시즌 IAP — 보조 매출

구독을 메인, IAP는 **결제 절벽 순간만**:

- 수능 D-30 패키지 49,900 / 모의고사 D-7 19,900
- 6·9월 평가원 대비 29,900 / 여름·겨울방학 패키지 29,900
- 개인화 트리거: "오답 100개 누적 → 회수 패키지 6,900" / "수열 망각률 78% → 집중 회독 14일 9,900"

구독자도 추가 결제하는 매출. 듀오링고도 Duolingo English Test 별도 매출 보유.

### 7.4 한국 결제 변종 (필수)

- ✗ "월 9,900원 매달 빠져나감 + 자동결제 알림 없음" → 부모 거부감
- ✓ "콴다 패스 1개월 사용권" 식 / 연간 일시불 할인 강조
- ✓ **자동결제 OFF 기본**, 매번 명시적 갱신
- ✓ **카카오 알림톡** 결제 직전 명확 고지

토스·카카오 멤버십·스타벅스 모두 이 구조.

### 7.5 가격 결정 보류 — 첫 6개월 무료 베타

첫 6개월은 가격 결정 보류. 부모/학생 인터뷰로 결제 의향 측정 후 결정.

다음 60일 액션:
1. 본인 학원 베타 = **완전 무료** (학생 5~10명)
2. 6주 데이터 → 유료 전환 의사 측정
3. 부모 5명 인터뷰: "월 9,900 vs 9,900짜리 패키지 vs 학원 같이 부담" 중 친숙한 것 측정
4. 결과 보고 모델 확정

초기 과금 포인트는 "AI가 문제를 풀어준다"가 아니라 **내가 올린 문제와 인강 기록을 계속 회독시켜준다 + AI가 매주 약점 진단**이어야 한다.

## 8. MVP 범위

SparkClaw에는 “처음부터 전과목 AI 수능 플랫폼을 만들겠다”보다, **작게 검증하겠다**가 훨씬 좋아.

### MVP 1단계

```text
학생 가입
문제사진 업로드
AI 과목/단원/유형 태깅
오답 원인 선택/수정
1일/3일/7일/14일 복습 예약
오늘의 회독 3개 생성
회독 플레이
XP / 스트릭 / 기억 HP
```

### MVP 2단계

```text
인강명·단원명 입력
캡처+메모 회독화
오답던전
망각방어전
과목별 약점 리포트
독학재수 관리 대시보드
```

### MVP 3단계

```text
수능 유형 카드 RAG 고도화

순공리그
```

## 9. 법무·보안 포지션

이 부분도 SparkClaw에서는 오히려 신뢰 포인트로 써야 해.

순공대장은 원본 문제사진과 인강 캡처를 많이 쌓는 서비스가 아니라, **원본 데이터는 최소 저장하고 파생 학습 데이터만 누적하는 구조**로 설계되어 있다. 데이터는 Raw Data와 Derived Data로 나누고, 원본 이미지·문제 원문·인강 캡처는 최소 저장/자동 삭제하며, 과목·단원·유형·오답 원인·복습카드 같은 파생 데이터만 장기 자산화하는 방향이다. fileciteturn4file2

사업소개서 문장으로는 이렇게 쓰면 된다.

> 순공대장은 문제 원문과 인강 캡처를 대량 저장하는 문제은행이 아니라, 학생의 개인 학습 입력을 분석해 복습 가능한 파생 학습 객체로 전환하는 AI 회독 엔진입니다. 원본 데이터는 최소 저장하고, 과목·단원·유형·오답원인·복습시점 등 파생 데이터 중심으로 관리해 저작권·개인정보 리스크를 줄입니다.

## 10. SparkClaw 적합성

SparkClaw는 1인 또는 소규모 AI 창업자를 대상으로 하고, 초기 투자 5천만 원~2억 원, OpenAI·Anthropic·GCP·Azure 등 AI 인프라 혜택을 제공한다고 안내하고 있다. 또한 서류 스크리닝에서는 학력·경력보다 AI를 팀원처럼 운용한 방식을 본다고 설명한다. ([sparkclaw.co.kr](https://www.sparkclaw.co.kr/))

그래서 순공대장은 이렇게 포지셔닝하면 좋다.

> 순공대장은 창업자 한 명이 AI Agent를 팀원처럼 운용해 만드는 AI-native 교육 제품입니다. 제품 내부에도 16개 Agent가 문제사진 분석, 수능 유형 RAG, 난이도 조절, 회독 스케줄링, 게임 리텐션을 담당합니다. 즉, SparkClaw가 찾는 “AI를 팀원으로 쓰는 창업자”와 “AI Agent가 실제 제품의 핵심인 서비스”가 동시에 맞물리는 프로젝트입니다.

### Agent 운용 방식 (듀얼 트랙)

순공대장 창업자는 1인이지만 운용 팀은 두 개의 Agent 트랙으로 구성됩니다.

**Product Agents (제품 안, 사용자에게 작동)**: 16개 Agent로 구성된 학습 분석 엔진. Input Router → OCR → Question Analysis → Type RAG → Variation → Solver → Review Scheduling → Quest → Game 파이프라인. MVP 1차에 9개를 구현하고, MVP 1.5/2차에 나머지 7개로 단계적 확장합니다.

**Development Agents (개발 운용, Mike + Claude Code 협업)**: 50+ subagent와 skill로 구성된 가상 개발 팀. oh-my-claudecode(전략/구현/디자인/리뷰), superpowers(brainstorming/TDD/verification), vercel(AI SDK/배포), feature-dev(아키텍처), design-system(UI 일관성), arch-guard(FSD 검증) 등을 1인 창업자가 팀처럼 운용합니다.

이 듀얼 구조는 **"AI Agent가 AI Agent를 만든다"**는 메타-레벨로, 1인 창업이 8주 안에 16-Agent 학습 엔진을 동작시키는 것을 가능하게 합니다. 자세한 트랙 분리와 디스패치 매트릭스는 `docs/agent-strategy/2026-05-14-agent-tracks.md` 참조.

수치로 요약:
- Product Agents: **16개** (백서) / **9개** (MVP 1차 구현)
- Development Agents: **50+ subagent / skill**
- 인간: **1명** (창업자)
- 개발 기간: **8주** (MVP 1차)

### 10.1 부트캠프 통과를 위한 60일 우선순위 (전략정리 §7.3)

| 우선순위 | 액션 | 비고 |

| 🔥 1 | **베타 사용자 데이터** — 본인 학원 학생 10~20명, 2주만 돌려도 됨. 종이 + Notion + 데모 앱 혼합 OK | "이런 학생이 이렇게 썼다" 스토리 + 망각률/정답률 숫자 1개 + 인터뷰 2~3개. **Deck hero slide.** |

# SparkClaw 제출용 1000자 소개문 초안

순공대장은 수능생을 위한 듀오링고형 AI 회독 앱입니다. 한국 수능생은 인강과 문제풀이에 많은 시간을 쓰지만, 실제 문제는 공부한 내용을 다시 꺼내지 못하고 오답을 회수하지 못한다는 점입니다. 순공대장은 학생이 문제사진, 인강기록, 캡처, 메모를 올리면 AI Agent가 과목·단원·유형·오답원인·난이도를 분석하고, 1일/3일/7일/14일 복습 퀘스트로 전환합니다. 학생은 마스코트 캐릭터 ‘순공이’(가안, 듀공 형태)가 안내하는 오늘의 회독퀘스트, 오답던전, 망각방어전을 클리어하며 XP, 스트릭, 기억 HP, 순공리그를 통해 매일 복습 루틴을 유지합니다.

이 제품의 핵심은 문제풀이 AI가 아니라, 학생이 실제 공부한 흔적을 장기기억으로 전환하는 Source-to-Quest Engine입니다. 내부는 OCR, 문항분석, 수능 유형 RAG, 학생 기억 RAG, 난이도 조절, 회독 스케줄링, 게임 리텐션 Agent로 구성됩니다. 원본 문제와 인강 캡처는 최소 저장하고, 과목·단원·유형·오답원인·복습시점 등 파생 학습 데이터 중심으로 누적해 저작권과 개인정보 리스크를 줄입니다.

국내 초·중·고 사교육비는 2025년 27.5조 원, 고등학생 사교육비는 7.8조 원 규모이며, 2026학년도 수능 지원자는 55만 명 이상입니다. 순공대장은 이 중 인강 중심 독학재수생과 오답관리가 약한 수능생을 초기 타깃으로 월 구독형 B2C 모델을 검증하고, 이후 독학재수학원·스터디센터용 B2B2C 관리 대시보드로 확장합니다.

# 유니크한 교육 앱 설계

> 원본 소스: `유니크한 교육 앱 설계.txt`  
> 분류: **프로젝트·사업·전략**

---

맞아. **오답회수 모드 UI가 진짜 히트 포인트**야.  
이건 그냥 기능 하나가 아니라, 순공대장의 **킬러 UX**로 봐야 해.

핵심은 이거야.

> **학생이 틀린 순간, 해설만 보여주고 끝내지 않고  
> “바로 비슷한 문제로 다시 맞히게 만드는 화면”**

이게 기존 AI 풀이 앱과 완전히 달라지는 지점이야. 내부 설계에서도 오답회수 모드는 단순 유사문제 추천이 아니라 **오답 풀이 → 오답 원인 진단 → 유사문항 베리에이션 → 다시 맞힐 때까지 회독** 구조로 잡혀 있어. fileciteturn3file19

## 왜 UI가 히트냐

학생이 문제를 틀렸을 때 보통 흐름은 이거야.

```text id="zjrl2h"
틀림 → 해설 봄 → 이해한 것 같음 → 끝
```

근데 순공대장은 이렇게 바꾸는 거지.

```text id="kdzdvu"
틀림
→ 왜 틀렸는지 진단
→ 바로 비슷한 문제 1개 출제
→ 맞히면 오답회수 성공
→ XP / 숙련도 / 망각위험 감소
```

이게 수험생 입장에서 강력해.

**“아 내가 진짜 이해했는지 바로 확인된다”**  
이 감각이 생기거든.

## 오답회수 UI는 이렇게 가야 함

### 1. 틀린 직후 화면

```text id="bpw5tm"
아직 회수 안 됐어.

오답 원인
n과 n+1 인덱스 혼동

짧은 해설
aₙ₊₁을 구할 때 n=1을 넣으면 a₂가 됩니다.

[비슷한 문제로 바로 회수하기]
```

여기서 버튼 문구가 중요해.

좋은 CTA:

```text id="ln9wkz"
비슷한 문제로 회수하기
오답 바로 잡기
이번엔 맞혀보기
3분 안에 회수하기
```

별로인 CTA:

```text id="ie3s9z"
유사문제 풀기
추가 학습하기
다음 문제
```

“유사문제”는 기능명이고, **“회수하기”는 감정이 있음.**

## 2. 오답회수 플레이 화면

```text id="sk95a9"
오답회수 1/3

이번엔 숫자만 바꿨어.
진짜 이해했는지 확인해보자.

[문제 영역]

[힌트 보기] [정답 제출]
```

상단에 반드시 들어갈 것:

| UI 요소 | 이유 |
|---|---|
| 오답회수 1/3 | 짧게 끝난다는 안정감 |
| 오답 원인 칩 | 왜 이 문제를 푸는지 명확 |
| 변형 단계 표시 | 숫자변형 / 조건변형 / 표현변형 |
| 순공이 한마디 | 응원감 + 캐릭터성 |
| 예상 시간 | 피로감 감소 |

예시:

```text id="ebo0f0"
오답회수 1/3
수학 · 수열 · 점화식
오답원인: 인덱스 혼동
변형: 숫자만 변경
예상 2분
```

## 3. 맞혔을 때 결과 화면

```text id="d33epv"
오답회수 성공!

+30XP
점화식 숙련도 +3
망각위험 높음 → 보통
다음 회독: 3일 후

순공이:
“좋아. 이번엔 진짜 회수했다.”
```

이 화면은 보상감을 줘야 해.  
순공대장의 핵심은 “공부했다”가 아니라 **“오답을 회수했다”**는 성취감이니까.

## 4. 또 틀렸을 때 화면

```text id="y049x8"
아직 흔들린다.

이번엔 더 짧게 간다.

OX
점화식에서 aₙ₊₁은 다음 항을 의미한다. O / X
```

이게 좋아.  
계속 비슷한 난이도 문제를 던지면 짜증나니까, 틀리면 **L1 개념 회수**로 내려가야 해. 기존 난이도 설계에서도 2회 연속 오답 시 개념 카드로 회귀하고, 최근 반응을 보고 난이도를 조절하는 구조가 맞다고 잡혀 있어. fileciteturn3file2

## 핵심 UI 구조는 이거야

```text id="ur8fab"
[틀림]
↓
[오답 원인 카드]
↓
[회수 CTA]
↓
[가까운 변형 문제]
↓
[성공/실패 결과]
↓
[다음 회독 예약]
```

## 홈에도 오답회수 카드가 보여야 함

홈 화면에 이렇게 하나 박히면 강해.

```text id="3e4v3f"
오늘 회수할 오답

수학 · 점화식
7일 전 틀림
오답원인: 인덱스 혼동
망각위험 높음

[오답회수 시작]
```

이건 “오늘의 회독”보다 더 강한 동기야.  
학생 입장에서는 **내가 틀린 걸 다시 잡는다**는 감각이 명확하니까.

## 이 UI의 진짜 이름

나는 이걸 순공대장의 핵심 순간으로 정의하고 싶어.

> **Correction Moment**  
> 학생이 “틀렸다”에서 끝나지 않고 “다시 맞혔다”로 전환되는 순간.

한국어 제품 언어로는:

> **오답회수 순간**

이게 히트 포인트야.

## 최종 판단

**오답회수 모드는 탭 하나가 아니라, 순공대장의 핵심 플레이 루프**로 넣어야 해.

하단 탭에 “오답회수”를 따로 크게 박기보다는:

```text id="el9ke3"
홈: 오늘 회수할 오답 카드
회독 플레이: 틀린 직후 회수 모달
회독 탭: 오답회수 리스트
결과 화면: 오답회수 성공 보상
통계 화면: 오답회수율
리그 점수: 오답회수 점수
```

이렇게 전 앱에 깔아야 해.

한 줄로 정리하면:

> **순공대장의 히트 UI는 “문제 맞히기”가 아니라 “틀린 문제를 다시 맞히는 순간”이다.**

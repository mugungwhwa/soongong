## 0. 한 줄 정의 (재확인)

> **콴다는 학생의 막힘을 풀고, 순공대장은 학생의 까먹음을 풉니다.**

기능 비교가 아니라 **무엇을 학습이라고 믿는가**의 차이.

## 1. 교육 철학 — 콴다 vs 순공대장

### 1.1 두 모델의 본질적 차이

| 차원 | 콴다 (1세대) | 순공대장 (2세대) |
|---|---|---|
| 학습 정의 | 막혔을 때 즉시 답 | 풀고 나서 다시 풀게 함 |
| 학습학적 전제 | 막힘은 학습의 적 | 인출(retrieval)이 학습 |
| 학생의 역할 | 소비자 (AI가 가르침) | 수행자 (AI가 회수 강제) |
| 시간 가치관 | 시간을 아끼는 것 | 시간을 들이는 것 |
| 결제 동기 | "막혔을 때 빠른 해결" | "오를 점수를 위한 투자" |
| 부모 인식 | 공부 도구 | 성적 관리 시스템 |
| 학원과 관계 | 학원 대체재 (위협) | 학원 보완재 (협업 가능) |

### 1.2 같은 학생, 같은 문제, 다른 결과

**콴다 시나리오**: 문제 사진 → 풀이 → "이렇게 푸는 거구나" → 닫음 → 3일 뒤 같은 유형에서 또 막힘 → 또 검색. **반복적 무력감이 의존을 만듦** (비즈니스로는 완벽).

**순공대장 시나리오**: 문제 사진 → 오답 원인 진단 + 짧은 해설 → V0~V5 변형으로 회수 → 1/3/7/14일 자동 회독 → 3개월 뒤 같은 유형 풀어냄. **학생이 졸업하면 떠나지만 점수가 오름**.

### 1.3 학술적 뒷받침 (Deck 한 줄)

> **Karpicke & Roediger (2008, *Science*)**: 단순 학습 그룹은 1주 뒤 80%를 까먹었고, 인출 연습 그룹은 1주 뒤 80%를 기억했다. **같은 시간, 4배 차이.**

이게 순공대장이 가져갈 수 있는 가장 무거운 학술적 모트. 한국 에듀테크가 본격적으로 활용 안 한 영역.

### 1.4 콴다 포지셔닝 narrative (Deck용)

심사위원 질문 "왜 콴다가 아닌가요?" 대비 답변:

> "콴다는 한국 학생의 '막힘'을 풀어 1억 명을 모았습니다. 우리는 콴다가 풀지 않은 다음 문제 — '까먹음' — 을 풉니다. 한국 학생이 매일 8시간 공부하고도 점수가 안 오르는 이유는, 답을 못 찾아서가 아니라 푼 걸 기억하지 못해서입니다."

콴다 = 1세대 (질문 → 답), 순공대장 = 2세대 (풀이 → 기억). **시장이 1세대를 흡수했으니 2세대로 갈 때.**

## 2. 듀오링고 차용의 진짜 의미

### 2.1 UI가 아니라 메커니즘 이식

듀오링고 = **인출연습 + 분산학습 + 게임화의 가장 성공한 구현체**. 순공대장은 그걸 한국 수능에 정확히 이식.

| 듀오링고 | 순공대장 |
|---|---|
| 매일 5분 단어 회상 | 매일 10분 오답 회수 |
| 까먹기 직전 알림 | 망각률 기반 회독 알림 |
| 스트릭 (연속 학습) | 스트릭 + 순공리그 |
| 단어 강도 (Strength) | 기억 HP |
| 레슨 트리 세로 경로 | 회독길 세로 경로 |
| 틀린 단어 다음 레슨에 다시 | V0~V5 변형으로 회수 |
| 골드 → 새 레슨 잠금해제 | XP/등급 → 4점보스 |

### 2.2 듀오링고의 그림자 — 피해야 할 함정

**"5년 했는데 영어로 대화는 못 한다"** 비판. 게임화가 강해져서 점수/스트릭 추구가 실제 학습을 잠식.

순공대장의 방어 원칙:

1. **게임화는 "행동 유도"용, "성과 측정"용 아님**
   - 스트릭, XP, 등급 → 매일 돌아오게 만드는 도구
   - 진짜 성과 지표 = 망각률 감소 + 모의고사 점수
   - 부모 리포트에 둘을 명확히 분리

2. **L5 보스전 = 실전 연결 메커니즘**
   - 듀오링고 약점: 진짜 대화 시뮬레이션 부족
   - 순공대장: L1~L3 회독 → L5 보스에서 통합 출현
   - 게임화와 실전을 연결

3. **월 1회 모의고사 모드**
   - 실제 수능 형식, 시간 제한, 점수 채점
   - 게임 점수가 아닌 진짜 점수 추적

### 2.3 Deck 포지셔닝

**나쁜 답**: "한국적으로 현지화한 듀오링고" → 카피캣

**좋은 답**:
> "듀오링고는 언어학습 retention 문제를 가장 성공적으로 푼 회사입니다. 우리는 메커니즘을 베낀 게 아니라 **같은 인지심리학을 다른 도메인에 적용**했습니다. 한국 수능은 듀오링고보다 한 단계 어려운 게임입니다 — 단순 회상이 아니라 복합 문제 풀이이기 때문입니다. 그래서 우리는 듀오링고에 없는 **L1~L5 난이도 계층 + 변형 축 + 보스전**을 추가했습니다."

**한국판 듀오링고가 아니라 듀오링고의 다음 진화** → valuation이 달라짐.

## 3. 회독 엔진 — 진짜 모트

### 3.1 사업의 본질 — "엔진 회사" 정체성 확정

| 길 | 의미 | 결정 |
|---|---|---|
| A. 컨텐츠 회사 | 우리가 수능/공시 문제 만든다 또는 라이센싱 | ❌ 폐기 |
| B. 엔진 회사 | 우리는 회독 엔진만 만든다. 콘텐츠는 사용자 자료 | ✅ 채택 |

**확정 이유**:
- 확장 속도 (컨텐츠 라이센싱은 시장당 6~12개월, 엔진은 자동)
- 법무 리스크 회피 (기출 저작권 우회)
- Anki 30년 검증된 모델
- 데이터 차별화 (사용자가 올릴수록 메모리 RAG 진해짐)
- Mikhoon이 컨텐츠 전문가가 될 필요 없음

**현재 Source-to-Quest Engine 설계 그대로 다른 버티컬 확장 가능.**

### 3.2 진짜 모트 — 데이터/RAG/온톨로지

**겉으로는 회독앱 다 비슷해 보임. 실제 작동 여부는 데이터 레이어 1cm 아래에서 결정.**

엔진이 답해야 할 어려운 질문 (예: 수열 단원 오답 시):
- 이 문제가 점화식 추론인가 수렴값 계산인가? (Question DNA)
- 학생이 개념을 모르는가, 계산 실수인가, 조건 해석 실수인가? (오답원인)
- L1 ↔ L5 사이 어디로 보낼까? (Difficulty Agent)
- 같은 단원이지만 건드린 적 없는 변형 축은? (Variation Agent)
- 학생의 현재 망각률과 어떻게 맞물리나? (Student Memory)

### 3.3 온톨로지 구조 (트리)

```
과목 → 영역 → 단원 → 개념 → 유형 → 풀이전략 → 오답패턴
                                              ↓
                                        변형축 (조건/수치/제시방식)
```

이게 정확하지 않으면 "비슷한 문제"가 랜덤 문제가 됨 → 한 번 풀어보고 "도움 안 되네" → 이탈.

### 3.4 학생 메모리 RAG — 복리 자산

```
1주 차: 학생 A가 "수열 점화식" 틀림 → 단순 태그
1개월: A의 조건 해석 패턴, 계산 실수 빈도, 무너지는 형태 데이터
3개월: A의 풀이 사고 모델 형성
6개월: A와 유사 사고 패턴 학생 B 데이터로 선제적 추천 가능
```

**개별 학생을 깊게 알수록 회독 정확 → 학생 머무름 → 데이터 축적**. ML flywheel. 콴다/Anki가 못 만드는 자산.

### 3.5 진단평가 + 사전확률 — Onboarding 핵심

베이지안 표현:

```
P(학생 실력 분포 | 진단평가 결과)
  ∝ P(진단평가 결과 | 학생 실력) × P(학생 실력)
                                      ↑
                                  Prior (페르소나/온톨로지)
```

**Prior가 정확할수록 진단평가 문항 수 감소**:
- 약한 prior → 50문제 → 1시간 → 90% 이탈
- 강한 prior → 10문제 → 5분 → onboarding 통과

학습 도메인의 어려움:
1. 페르소나가 다차원 (단원 × 유형 × 풀이패턴 × 망각률)
2. 시간변동성 (매일 변함 — 망각 + 학습)
3. 상호의존성 (수열 ↔ 극한 ↔ 함수)

→ **계층적 베이지안 모델** 필요. Mikhoon 백그라운드와 정확히 매칭.

### 3.6 콜드 스타트 해법

신규 가입 학생은 데이터 0 → 추천 부정확 → 이탈 → 데이터 축적 시작 못함.

**Bootstrap 데이터 전략**:
- 가입 시 진단 평가 10문제 → 강한 prior 확보
- 같은 학원/학년의 다른 학생 패턴을 기본값
- 첫 1주는 인기 회독 코스부터 (개인화 약하게, 보편 도움)

게임의 onboarding 튜토리얼 설계와 동일. **첫 7일 retention이 LTV의 90%**.

### 3.7 RAG Evaluation — 진짜 어려운 부분

"이 변형 추천이 좋은 추천이었나?" 측정 난점:
- 학생이 맞히면 좋은 추천? → 너무 쉬웠을 수도
- 학생이 틀리면 나쁜 추천? → 학습 도움 됐을 수도
- 학생이 끝까지 풀면 좋은 추천? → 의무감일 수도
- 망각률 떨어지면 좋은 추천? → 측정 1개월 lag

**해법**:
- 짧은 metric (정답률, 풀이시간, 자신감) + 긴 metric (모의고사 점수) 병행
- 짧은 metric으로 빠르게 iterate
- **CUPED 기법 적용** (학생별 사전 실력으로 variance 줄임) — 게임업계 A/B 테스트와 동일

### 3.8 온톨로지의 "회색지대" — 결정 사항

수학 문제 하나가 "수열 + 극한 + 함수의 연속"에 동시 걸려있을 때:

**Anki 30년의 교훈 적용**:
- 초기 — **단일 분류 + 사용자 수동 다중 태그 가능**
- AI는 단일 추천만, 사용자가 보강
- 1년 데이터 쌓이면 다중 분류 자동화

### 3.9 데이터 아키텍처 결정 사항

**하이브리드 모델**:
- **Postgres** = truth (온톨로지 트리, 사람이 검수 가능한 구조)
- **pgvector** = 검색용 (학습 객체 임베딩, 의미 기반 유사도)

→ **P3 sub-plan에 명시적으로 박혀 있는지 확인 필요**.

## 4. 버티컬 확장 — 회독은 공부의 본질

### 4.1 회독의 보편성

인지심리학 100년 결론: 인출연습 + 간격반복은 **시험 종류와 무관하게 작동**.

한국 시장은 버티컬별로 분절됨 (각자 자기 시장에서만 놂):
- 수능: 콴다, 매스프레소, 설탭
- 공시: 에듀윌, 해커스
- 자격증: 에듀윌, 해커스, 시대고시
- 의대/약대: 메디프리뷰

**공통 회독 엔진 = 이 분절을 가로지르는 자산.**

### 4.2 버티컬 매력도

| 시장 | 응시생/년 | 결제력 | 학습 강도 | 회독 적합도 | 우선순위 |
|---|---|---|---|---|---|
| 수능 | 55만 | 부모 지갑 (강함) | 매우 높음 | 최상 | **1년 차** |
| 공무원시험 | 20만 | 본인 (강함) | 매우 높음 | 최상 | **2년 차** |
| 공인중개사 | 30만 | 본인 (강함) | 높음 | 상 | 검토 |
| 자격증 (산업기사~기사) | 100만+ | 본인 (중) | 중 | 중 | 검토 |
| 어학 (토익/토플) | 200만+ | 본인 (약~중) | 중 | 중 | 후순위 |
| 전문직 (의/약/변) | 5천~3만 | 본인 (매우 강) | 극도로 높음 | 최상 | 검토 |

### 4.3 확장 순서 (확정안)

```
1년 차: 수능 집중 + 검증
        └ 본인 학원 베타 → 알고리즘 검증 → 첫 매출
2년 차: 공시 확장
        └ 같은 엔진, 새 온톨로지
3년 차+: 자격증 → 어학 → 글로벌
```

이 순서는 **Mikhoon의 도메인 지식이 누적되는 방향**. 처음부터 호리즌탈로 가면 진짜 어려움.

### 4.4 각 버티컬마다 필요한 작업

엔진은 같지만 **OCR/태깅 fine-tuning + 온톨로지는 새로 만들어야 함**:
- 수능: 수학 점화식, 영어 지문, 국어 문학
- 공시: 행정법 판례, 한국사 연표, 영어 단어장
- 자격증: 도면, 회계 분개, 법조문

**이게 진짜 모트**. 엔진은 카피되지만 **버티컬별 튜닝된 OCR + 도메인 RAG**는 1~2년 걸림.

### 4.5 온톨로지 제작 전략

과목당 최소 3개월 전문가 시간 필요 (단원 30~50 / 개념 200~500 / 유형 1,000~3,000 / 변형 축 3~5).

**채택**: B + D 하이브리드
- 평가원 발간 학습목표 + 기존 출제 분석 → LLM이 정리 → 본인 학원 멘토가 검수
- 단원당 1~2주

## 5. 비즈니스 모델 — 듀오링고형 하이브리드

### 5.1 모델 선택 — IAP 단독에서 하이브리드로

초기 검토안인 **IAP 단독 모델은 폐기**. 듀오링고 FY2025 데이터를 학습한 결과, **구독 (메인) + IAP 시즌 (보조) + B2B2C (안정)** 3중 하이브리드가 valuation/안정성/확장성에서 모두 우위.

핵심 이유:
- **Valuation 멀티플**: 구독 SaaS는 매출의 6~10배 평가, IAP는 2~4배
- **MRR 예측 가능**: IAP의 "수능 시즌만 매출 폭발" 변동성 회피
- **AI 차별화 가격 인상 명분**: 듀오링고 Max가 증명한 모델
- **글로벌 확장 표준**: 일본/미국 진출 시 듀오링고 모델이 익숙

### 5.2 듀오링고 모델 학습 (FY2025 기준)

**최신 매출 구조**:
- 2025년 연 매출 약 **$1.03B (회사 첫 10억 달러 돌파)**
- EBITDA 마진 **29.5%**
- 유료 가입자 **11.5M**, 유료 침투율 **9%** (LTM MAU 기준)
- 매출 구성: **구독 83% + 광고 13% + IAP/기타 4%**
- ARPU 약 **$6/월** 수준

**4단 무기**:

1. **Freemium 깔때기 정교함**
   ```
   40+ 언어 무료 (광고) → MAU 1억+
     → Super (무광고 + 기본) → 유료 9% 전환
       → Max (AI 기능) → 프리미엄 5% 추가
         → Family Plan → 가구 단위 lock-in
   ```
   핵심: **무료가 진짜로 쓸 만함**. 6개월 쓰다가 광고 거슬려서 결제.

2. **AI 차별화로 ARPU 인상**
   - Max 플랜: GPT-4 기반 Roleplay, Video Call (Lily와 대화), Explain My Answer
   - 같은 사용자에게 더 받는 게 신규 모으는 것보다 5배 효율
   - **2년 전엔 못 받았을 가격을 AI 덕에 받음**

3. **Family Plan = 가구 매출 잠금**
   - 한 명 결제로 가족 5명까지
   - 해지 어려움 (한 명 끊으면 가족 다 끊김)
   - 한국 입시 가구 (형제자매) 완벽 적합

4. **매출 안정성**
   - 구독 매출 = 매월 안정적 캐시플로
   - 시즌 변동성 없음

### 5.3 한국 시장 적용 — 구독 변종

이전 결론: "한국 부모 = 구독 거부감". 정정:

- ✗ "월 9,900원 매달 빠져나감 + 자동결제 알림 없음" → 거부감
- ✓ "콴다 패스 1개월 사용권" → 잘 팔림
- ✓ "스터디언 연간 멤버십" → 잘 팔림

즉 **"구독" 자체가 문제가 아니라 "자동결제 + 매달 무알림"이 문제**.

한국에서 작동하는 구독 변종:
- **연간 일시불** (할인 강조)
- **시즌 패스** (수능까지 1년치)
- **자동결제 OFF 기본**, 매번 명시적 갱신
- **카카오 알림톡**으로 결제 직전 명확 고지

→ 토스, 카카오 멤버십, 스타벅스 모두 이 구조.

### 5.4 순공대장 티어 구조 (확정안)

| 티어 | 가격 | 핵심 가치 | 타겟 |
|---|---|---|---|
| **Free** | 0원 (광고) | 일일 회독 3개, 망각방어전 주 1회, 오답 50개 한도 | 신규 유입 깔때기 |
| **Super 순공** | 월 9,900 / 연 79,000 | 무제한 회독, 오답 무제한, 광고 제거 | 메인 유료층 |
| **순공 Max** | 월 19,900 / 연 159,000 | + AI 멘토, 무한 변형 문제, 부모 리포트 주간 | 진지한 입시생 |
| **가족 플랜** | 연 199,000 (형제 3명까지) | 가구 단위 결제 | 형제자매 입시 가구 |
| **B2B2C** | 학원당 별도 | 학원 대시보드 + 학부모 리포트 | 학원/관리형 |

**시즌 IAP는 보조 매출**로 별도 운영 (5.6 참조).

### 5.5 Max 티어 AI 차별화 (가격 인상 명분)

듀오링고 Max의 천재성 = **AI 기능 자체가 가치 제안의 핵심**. 그냥 "무제한 회독"으로는 월 19,900 못 받음.

순공대장 Max 전용 AI 기능 (Claude API 활용):

- **AI 회독 코치**: 매주 학생 약점 분석 → "이번 주 가장 위험한 단원은 수열입니다" 자동 코스 추천
- **AI 풀이 분석**: 풀이과정 캔버스 → AI가 사고 흐름 분석 → "이 단계에서 막혔어요. 비슷한 함정 5개 더" 자동 생성
- **AI 부모 코치**: 학부모 카톡으로 주간 리포트 + "이번 주 자녀에게 이런 격려 한마디" 제안
- **AI 모의고사 분석**: 단순 점수 아니라 단원별/유형별/시간배분 진단

**이 기능들이 Max 가격을 정당화**. 동시에 Claude API 비용이 큰 변동비 → 매출 따라 자동 스케일.

### 5.6 시즌 IAP — 보조 매출 트리거

구독을 메인으로 두고, IAP는 **결제 절벽 순간만** 활용:

- **수능 D-30 패키지**: 49,900원 (30일 집중 코스)
- **모의고사 D-7 패키지**: 19,900원
- **6월/9월 평가원 대비**: 29,900원
- **여름방학 / 겨울방학 패키지**: 29,900원

이건 **구독자도 추가 결제하는 매출**. 듀오링고도 Duolingo English Test 별도 매출 보유.

순공대장만 가능한 개인화 IAP 트리거 (AI가 결제 모먼트 생성):
- 오답 100개 누적 시: "지금 회수 안 하면 까먹습니다 → 회수 패키지 6,900"
- 취약 단원 발견 시: "수열 망각률 78% → 집중 회독 14일 9,900"

### 5.7 매출 시뮬레이션 — 듀오링고 멀티플 적용

**시나리오: 한국 수능생 + 공시생 100만 명 사용자 (3~5년 그림)**

- Free 사용자 91만 (광고)
- Super 7만 × 월 9,900 = 월 6.9억
- Max 1.5만 × 월 19,900 = 월 3억
- 가족 플랜 5,000 가구 × 월 16,500 = 월 0.8억
- 시즌 IAP = 연 20억
- B2B2C 학원 50곳 × 월 200만 = 월 1억

**합계: 월 11.7억 + 시즌 IAP 연 20억 = 연 매출 약 160억**

EBITDA 29.5% 적용 시 영업이익 47억. **시리즈 B 가능 수치.**

근거: 한국 수능 응시자 55만 + 공시 20만 + 자격증 일부 = TAM 150만+. 도달 가능.

### 5.8 모델별 Valuation 멀티플 비교

| 차원 | 듀오링고형 (구독) | IAP 단건 |
|---|---|---|
| 매출 안정성 | 매우 높음 (MRR) | 낮음 (시즌 의존) |
| ARPU 성장성 | 높음 (Max 인상 가능) | 낮음 (가격 천장) |
| 한국 결제 친화도 | 중 (조정 필요) | 높음 |
| 30% 수수료 | 같음 | 같음 (외부결제로 일부 회피) |
| Lock-in | 강함 (가족, 연간) | 약함 |
| **Valuation 멀티플** | **6~10x 매출** | **2~4x 매출** |

같은 매출이어도 valuation 2~3배 차이. 듀오링고 시총 100억 달러는 구독 모델이 만든 것.

### 5.9 첫 6개월 전략 — 가격 결정 보류

**중요**: 첫 6개월은 가격 결정 보류. 무료로 베타 운영하면서 데이터 수집.

이유:
- 듀오링고도 처음 5년 매출 거의 없었음
- 한국 시장 PMF 검증 안 된 상태에서 가격 박으면 회귀 어려움
- 부모/학생 인터뷰로 결제 의향 측정 후 결정

다음 60일 액션:
1. 본인 학원 베타 = **완전 무료** (학생 5~10명)
2. 6주 데이터 → 유료 전환 의사 측정 (인터뷰)
3. 부모 5명 인터뷰: "월 9,900 vs 9,900짜리 패키지 vs 학원 같이 부담" 중 가장 친숙한 것
4. 결과 보고 모델 확정

### 5.10 게임 IAP vs 학습 결제 — 정서 차이 (유효)

게임은 결제가 즐거움, 학습은 결제가 부모 압박.

원칙:
- "즐거운 결제"가 아니라 **"안심 결제"** 만드는 게 핵심
- "이번 패키지 안 사면 점수 떨어집니다" 같은 fear-based 카피 → 학부모 반발
- 대신 "이 코스 끝내면 모의고사 X점 상승 기대" → 가치 기반 카피
- Max 티어 카피: "AI가 자녀의 약점을 매주 진단합니다" → 부모가 사고 싶어함

## 6. 글로벌 확장 전략

### 6.1 핵심 인식 — "확장"이 아니라 "재창업"

**듀오링고가 글로벌 쉬웠던 이유** (순공대장에 없음):
- 영어 = 모든 시장 공통 학습 목표
- 콘텐츠 = 언어 자체, UI만 번역
- 평가 기관 표준화 (TOEIC, IELTS)

**시험 회독이 글로벌 어려운 이유**:
- 시험은 국가별로 완전 다름
- 콘텐츠 = 각 시장마다 처음부터
- 학습 문화 다름 (한국·일본 입시 vs 미국 holistic)

→ **글로벌 확장 = 각 시장에서 mini-순공대장 만들기.**

### 6.2 시장 우선순위

**1순위: 일본** (3~5년 차)
- 사교육 9조엔 (한국의 3배)
- 입시 문화 90% 한국과 동일
- 한자권 → 단어 차용 쉬움 ("회독" → "回読")
- **현실 모델**: 일본 학원 합작 진출, 일본 출신 공동창업자 필수

**2순위: 동남아** (5년 차)
- 베트남/태국/인도네시아
- 시장 작지만 성장률 15~20%/년
- 객단가 낮음 → 한국 검증 엔진을 SDK/라이센스로 제공 (B2B2C)

**3순위: 미국 Professional Exam** (5~7년 차)
- K-12는 어려움 (rote learning 문화 거부감)
- **MCAT, Bar, CPA, CFA** 등 시험준비 영역만 진입
- 사용자 적지만 단가 매우 높음 (월 $50~200)

**4순위: 중국** — 단기 포기 (雙減政策 규제)

### 6.3 "역방향 글로벌" — Niche 옵션

한국 시장 + 글로벌 사용자:
- **TOPIK** (한국어능력시험): 응시자 50만 명/년, 전 세계
- **한국 의대/대학 외국인 학생**
- 콘텐츠는 한국어 (이미 자산), 사용자 유입은 글로벌
- 한국 정부 K-콘텐츠 지원 가능성

작지만 시작점으로 매력적.

### 6.4 글로벌 narrative (Deck용)

```
1~3년: 한국 수능 + 공시 + 자격증 → 도메인 모트
3~5년: 일본 입시 → 가장 유사한 시장
5년+: 글로벌 Professional Exam → 단가 최고
```

**중요**: 첫 18개월은 한국 수능에만 집중. 너무 빨리 글로벌 시도하면 양쪽 다 망함 (Riid, 콴다 글로벌 실패 사례 참조).

### 6.5 브랜드명 정책

- **글로벌 브랜드**: **Soongong** 그대로 (Duolingo, Anki, Kakao 모두 의미 없는 단어로 글로벌)
- **영문 부제**: "The Recall Engine for Test Prep"
- **회사명**: Soongong Inc. 또는 별도
- **현지 마케팅 설명**: "Korean SAT Recall Engine" / "Japan College Exam Recall Engine"

리브랜딩은 PMF 검증 후 결정 (Facebook → Meta, Square → Block 사례).

### 6.6 "순공"의 영문 직역 (참고)

- **Net Study Time** (회계적 직역)
- **Deep Study Time** (Cal Newport "Deep Work" 차용)
- **Focused Study Time** (일반적)

브랜드명 변경하는 게 아니라, **영문 설명/마케팅 카피용**.

## 7. SparkClaw 통과 전략

### 7.1 SparkClaw 2026 조건

- 최대 **1억 초기 투자** + OpenAI/Anthropic/AWS 등 **5억 상당 크레딧**
- 프로세스: 온라인 신청 → 서류 → 부트캠프 → 최종 심사
- 통과 실패 시: Stage 2/3 풀로 분류, 후속 배치 우선
- 강조 키워드: **빌더 (실행 결과)**, **AI를 팀원으로**, **PMF**

### 7.2 순공대장의 매칭도 평가

| SparkClaw가 보는 것 | 순공대장 상태 |
|---|---|
| AI-native 1인 창업자 | ✓ 100% 오너 |
| 빌더 (실행 결과) | ✓ 8주 MVP 진행 중 |
| AI를 팀원으로 | ✓ 50 dev agent + 16 product agent |
| 명확한 시장 | ✓ 사교육 27.5조 + 공시/자격증 확장 narrative |
| 차별화/모트 | ✓ Source-to-Quest Engine, 학생 메모리 RAG |
| Defensibility | ✓ 도메인 + 온톨로지 |
| **Traction** | ⚠️ **약점 — 실사용자 데이터 0** |

### 7.3 부트캠프 통과를 위한 60일 우선순위

**1순위 — 베타 사용자 데이터 (최우선)**

본인 학원 또는 베타 채널 학생 10~20명. **2주만 돌려도 됨**. 종이 + Notion + 데모 앱 혼합 OK.

필요한 산출물:
- "이런 학생이 이렇게 썼다" 스토리 2~3개
- 망각률 변화 또는 정답률 변화 숫자 1개
- 학생 인터뷰 영상/녹취 2~3개
- 운영 후기

**deck의 hero slide가 됨**. 대부분 지원자가 "이거 만들 거예요" 단계인데, 순공대장은 "이미 쓰고 있어요" 단계 가능.

**2순위 — 포지셔닝 한 줄**

현재 "문제를 풀어주는 AI가 아니라 다시 풀게 만드는 AI" 그대로 유지. 단, **"수능 회독 앱" → "회독 리텐션 엔진"** 으로 격상.

**3순위 — AI-native 빌딩 시연**

50 agent 운용 그림을 5분 영상으로. SparkClaw가 보고 싶어하는 정확한 콘텐츠.

### 7.4 1억으로 2~3명 팀 — 현실 점검

| 구성 | 1년 비용 |
|---|---|
| 시니어 풀스택 1명 | 8,000만~1.2억 |
| 주니어 1명 | 5,000만~7,000만 |
| 디자이너 1명 | 5,000만~7,000만 |
| 4대보험 + 운영비 | +20~30% |
| **정직원 2~3명 1년** | **1.5억~2.5억** |

→ **정직원 2~3명 1년 불가능**. 가능한 옵션:

**옵션 A**: 정직원 1명 + 파트타임/외주 2명
**옵션 B**: 지분 공동창업자 1명 + 외주 (시리즈 A 대비 추천)

- 창업자 지분 80%+
- 실패 확률 30~40%

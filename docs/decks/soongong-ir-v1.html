망각곡선의 현실 추정

에빙하우스 망각곡선상 학습 후 복습이 없으면 기억은 하루~며칠 내 급격히 소실된다. 한 번 푼 문제도 회독 없이는 시험장에서 사라진다.

기존 도구의 한계 가정

콴다 등 AI 풀이는 "막힘"(지금 안 풀리는 문제)을 푼다. 하지만 "까먹음"(한 번 풀었으나 잊은 것)은 아무도 관리하지 않는다.

결국 새는 건 순공시간 가정

같은 개념을 반복 재학습하느라 순공시간이 샌다. 학생은 "공부한 만큼 남지 않는다"고 느낀다.

핵심 원칙
우리는 절대 학생이 모르는 걸 풀어주지 않는다. 독학에 필요한 리텐션 보존에 집중한다. (콴다 = 막힘 풀이 / 순공대장 = 까먹음 방지)

Section 04 · Why Now

독학재수관 확산 · 혼공 확대 추정

독학재수관(독학재수 전문 학원)이 빠르게 늘고 있다 — 수만휘 가맹점·수능선배 등 프랜차이즈 확산. 혼자 공부하지만 관리·리텐션이 필요한 학생군이 커지는 중이다.

인프라·콘텐츠 평준화 추정

인강·문제은행 등 교육 콘텐츠 접근성이 평준화됐다. 누구나 같은 자료를 본다.

AI OCR 성숙 추정

손글씨·수식 인식 OCR과 AI가 실사용 수준에 도달 — "내 문제 캡처"가 비로소 가능해졌다.

인프라 차이가 사라지면, 차별화는 순공시간으로 옮겨간다. 그리고 순공시간의 적은 망각이다.

Section 12 · 기술 ②
약점 정조준 변형 문항 생성 — 핵심 차별점
남의 문제집이 아니라, 내 약점을 정조준한 문항
오답 1건 → 1:1 정조준 검증

유사문제 양산이 아니다. 학생이 틀린 그 약점을 겨냥해 1:1로 변형 문항을 생성한다.

난이도 = 데이터 기반 검증

get_target_difficulty RPC가 L1~L5를 결정. 생성 코드의 자체 난이도 산정 = 0.

3가지 출제 모드 — 데이터가 고른 난이도에 맞춰
rebuild 검증

기초 스캐폴딩. 한 단 아래로 내려 빈 개념을 다시 세운다. (L↓)

maintain 검증

동급·다른 맥락. 같은 난이도, 새 상황으로 정착을 굳힌다. (L=)

stretch 검증

한 단 도전. 충분히 또렷할 때 한 칸 위로 끌어올린다. (L↑)

self-check 품질 게이트: 생성된 문항이 의도한 약점·난이도·정답 정합을 만족하는지 자체 검증하고, 실패 시 재생성한다. 사람 손을 거치지 않아도 출제 품질을 지킨다.

  • 1년간 학생을 직접 관찰 — "풀어도 까먹는다"를 현장에서 발견했다(책상 위 가설이 아니다).
왜 우리가 이기나

리텐션 DNA × 교육 현장 경험

  • 게임에서 단련한 재방문·이탈방지 설계를 학습에 이식
  • "까먹음"이라는 교육의 미해결 문제를 정조준
  • 문제 정의가 데이터가 아니라 현장에서 나왔다
팀 빌딩 방향 — 2대 역할

이 인사이트 위에 데이터 역량을 쌓아 MOAT 엔진을 완성한다.

  • DE(데이터 엔지니어) — 회독 엔진 데이터 파이프라인 · 문제 인식(OCR/AI) 인입 · 스케줄링 인프라
  • Data Ops — 데이터 품질·라벨링·운영 · 학습 데이터셋/프로파일링 · 모델 학습·배포 운영(ML 운영)
핵심: MOAT는 코드가 아니라 리텐션을 아는 창업자 + 데이터 역량을 쌓는 팀에서 나온다. 가정 대표 구체 이력 수치는 추후 확정.
검증 게임 리텐션 경력·독학재수관 투자운영·1년 현장 관찰은 창업자 실경험 · 가정 DE·Data Ops 채용 계획 및 세부 이력 수치는 추후 확정

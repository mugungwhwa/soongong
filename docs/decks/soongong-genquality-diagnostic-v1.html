병목 3갈래

— 어디에 먼저 투자할지의 지도
(a) OCR 부재
실물 사진을 못 읽는다

Pass A(실 OCR)가 10건 전부 실패. 사진 → 수식 텍스트 변환 단계 자체가 없어, 실제 사용 경로(학생이 찍은 사진)가 통째로 막혀 있다.

증거: Pass A 0/10 — 픽스처 난이도와 무관하게 전멸.
(b) 전처리 부재
정제하면 통과할 걸 놓친다

입력을 깨끗이 정제(전처리)하면 파서가 통과할 케이스가 있다. 04번이 바로 그 증거 — Pass A 실패였지만 Pass B에서 성공했다.

증거: 04번 = Pass A 실패 → Pass B ✅ 성공(전처리 효과 실증).
(c) 파서범위 협소
단순형마저 못 잡는다

깨끗한 입력(Pass B)에서도 01·02·03·05·06 단순 점화식이 실패. 고난도 07~10은 애초에 범위 밖. 파서 커버리지가 너무 좁다.

증거: 단순형 5건 실패 + 범위밖 4건 = 파서가 잡는 형태가 협소.

# 순공대장 MVP1 잔여 화면 기획서
> 작성일: 2026-06-10 | 대상: 오답노트 + 캘린더 화면

## 화면 1: 오답노트 (Wrong Notes)

### 1. 목적
틀린 문제를 단원·위험도 별로 한눈에 파악하고, 즉시 재도전 또는 회독 큐에 추가해 망각엔진(P4)과 연결하는 화면.

### 2. 핵심 기능
1. **오답 카드 리스트** — 틀린 문제를 카드 단위로 나열. 문제 썸네일, 단원 태그, 틀린 횟수, 마지막 시도일, 위험도 배지 표시.
2. **필터 칩** — 단원 / 날짜 범위(최근 7일·1개월·전체) / 오답 유형(실수·개념·응용) 3축 필터. 중복 선택 가능.
3. **재도전 CTA** — 카드 당 "재도전 →" 버튼. 탭 시 해당 문제 풀기 플로우(별도 페이지, MVP1에서는 stub)로 진입.
4. **회독 큐 추가** — 카드 우측 스와이프 또는 롱프레스 → "회독 큐에 추가" 액션 시트. 추가 시 review_quests에 오늘 기준 1/3/7/14일 스케줄 자동 생성.
5. **빈 상태** — 오답이 0개일 때 마스코트 + "아직 틀린 문제가 없어요! 잘 하고 있어요 🎉" 카피.

### 3. 보여줄 데이터

| 필드 | 타입 | 출처 |
|---|---|---|
| `question_id` | uuid | wrong_answers 테이블 |
| `thumbnail_url` | string | 문제 원본 OCR 이미지 썸네일 |
| `unit_tag` | string | 온톨로지 트리 단원명 (예: "수열의 극한") |
| `wrong_count` | int | 해당 문제 오답 누적 횟수 |
| `last_attempt_at` | date | 마지막으로 틀린 날짜 |
| `risk_level` | enum: low·mid·high | 망각 위험도 (망각엔진 P4 산출값, MVP1은 rule-based fallback) |
| `memory_hp` | 0–5 int | 현재 기억 HP (게임성 기획 §4-2 기준) |
| `review_queued` | bool | 회독 큐 추가 여부 (추가 시 버튼 비활성화) |

**MVP1 fallback 가정**: P4 망각엔진 미완성 시 `wrong_count` ≥ 3 → high, 1-2 → mid, 처음 틀림 → low 로 rule-based 처리.

### 4. 사용자 액션

| 액션 | 결과 |
|---|---|
| 필터 칩 탭 | 선택 상태 토글, 리스트 즉시 필터링 (클라이언트 사이드) |
| 카드 탭 | 문제 상세 → 재도전 진입 (MVP1 stub) |
| "재도전 →" 버튼 탭 | 동일 |
| 우측 스와이프 | 회독 큐 추가 액션 노출 |
| 회독 큐 추가 탭 | 해당 문제 review_quests에 스케줄 삽입, 버튼을 "추가됨 ✓"로 전환 |
| 상단 정렬 드롭다운 | 최신순 / 위험도 높은순 / 틀린 횟수 많은순 정렬 |

### 5. 상태 처리

| 상태 | UI |
|---|---|
| **빈 상태** | 마스코트(`main.png`) + "아직 틀린 문제가 없어요" + "문제를 풀면 여기서 복습할 수 있어요" 서브카피. 배경 `bg-sunken`. |
| **로딩** | 카드 스켈레톤 3개 (pulse 애니메이션). 실제 카드와 동일 높이. |
| **에러** | "불러오지 못했어요" + "새로고침" 텍스트 버튼. 마스코트 표정 변화 없음(이미지 교체 없이 카피만으로 처리). |
| **필터 결과 없음** | "해당 조건의 오답이 없어요" + 필터 초기화 버튼. |

### 6. P4 연결점

- **망각엔진(P4)**: `risk_level`과 `memory_hp` 값을 P4 API에서 받아 카드 위험도 배지 색상에 반영. P4 미완성 시 rule-based fallback 사용.
- **review_quests**: "회독 큐에 추가" 액션이 `review_quests` 테이블에 `source: wrong_note` 플래그와 함께 1/3/7/14일 스케줄을 삽입. 캘린더 화면에서 해당 점 배지가 생성됨.
- **오답 누적 피드백 루프**: 재도전 후 맞히면 `wrong_count` 감소 + `memory_hp` 상승 → risk_level 갱신. 오답노트에서 실시간 반영.

## 화면 2: 캘린더 (Calendar)

### 1. 목적
1/3/7/14일 회독 스케줄(review_quests)을 월 단위 시각으로 파악하고, 날짜 탭으로 당일 할 것을 즉시 진입하는 화면.

### 2. 핵심 기능
1. **월 그리드 캘린더** — 월 이동(← →) + 오늘 날짜 강조(링). 날짜별 회독 차수 점 배지 표시.
2. **회독 차수 배지 색상 코딩** — 1일 회독=`mint-300`, 3일=`mint-500`, 7일=`mint-700`, 14일=`mint-900`. 한 날에 여러 차수 겹칠 수 있으므로 최대 3개 점 + 초과 시 "+N" 표시.
3. **이번 주 요약 스탯** — 캘린더 상단 스탯 바: "이번 주 완료 N개 / 예정 M개 / 연속 K일".
4. **날짜 탭 시 하단 시트** — 해당 날의 회독 리스트(과목·단원·차수·예상 소요시간). 각 항목 "시작하기 →" 버튼.
5. **마스코트 빈 상태** — 해당 날에 스케줄 없으면 "오늘은 쉬는 날이에요" + 마스코트.

### 3. 보여줄 데이터

| 필드 | 타입 | 출처 |
|---|---|---|
| `quest_id` | uuid | review_quests 테이블 |
| `due_date` | date | 회독 예정일 |
| `review_round` | enum: 1d·3d·7d·14d | 회독 차수 |
| `subject_name` | string | 과목명 (예: "수학") |
| `unit_name` | string | 단원명 (예: "수열의 극한") |
| `est_minutes` | int | 예상 소요 시간(분) |
| `status` | enum: pending·done·skipped | 완료 여부 |
| `source` | enum: upload·wrong_note | 퀘스트 생성 출처 |

**이번 주 스탯 계산**:
- `done_count`: `status = done AND due_date IN [이번 주 월~일]`
- `pending_count`: `status = pending AND due_date IN [이번 주]`
- `streak_days`: 연속 완료일 (오늘 기준 역산)

### 4. 사용자 액션

| 액션 | 결과 |
|---|---|
| ← / → 버튼 | 이전/다음 달 이동, 그리드 + 점 배지 갱신 |
| "오늘" 버튼 | 현재 월로 점프, 오늘 날짜 자동 선택 |
| 날짜 셀 탭 | 하단 시트 슬라이드업 (해당 날 퀘스트 리스트) |
| 하단 시트 "시작하기 →" | 회독 플로우 진입 (MVP1 stub) |
| 하단 시트 완료 체크 | `status = done` 업데이트, 점 배지 색상 dimmed + 체크 표시 |
| 하단 시트 외부 탭 / 스와이프 다운 | 시트 닫기 |

### 5. 상태 처리

| 상태 | UI |
|---|---|
| **빈 상태(월 전체)** | 그리드는 유지. 중앙 마스코트(`main.png`) + "아직 예정된 회독이 없어요" + "문제를 올리면 자동으로 일정이 잡혀요" 서브카피. |
| **날짜 탭 빈 상태** | 하단 시트 내 마스코트 + "오늘은 쉬는 날이에요 ☁️". |
| **로딩** | 그리드 셀 스켈레톤 (점 배지 자리만 pulse). 스탯 바 "--개" 플레이스홀더. |
| **에러** | 스탯 바 숨김, 그리드 중앙 "일정을 불러오지 못했어요" + 새로고침. |

### 6. P4 연결점

- **review_quests 생성**: 문제 업로드(P3) 완료 시 P4 망각엔진이 1/3/7/14일 스케줄을 `review_quests`에 삽입 → 캘린더에 점 배지 자동 생성.
- **오답노트 연결**: 오답노트에서 "회독 큐 추가" 시 `source: wrong_note` 퀘스트가 캘린더에 반영됨.
- **완료율 피드백**: `status = done` 누적 → 스트릭 카운터 + XP 지급. 캘린더가 게임화 루프의 시각적 보상판 역할.
- **망각 경고**: `due_date` 경과 + `status = pending` 퀘스트는 캘린더 점 배지에 경고 테두리(`risk-high` 색) 추가.

## 공통 UX 원칙

- 두 화면 모두 **모바일 우선**, 최대 폭 430px.
- 네비게이션: 하단 5칸 고정 네비 재사용 (오답노트 탭 = notebook 아이콘 활성, 캘린더는 today 탭 또는 별도 진입경로 — MVP1에서는 오늘 화면 내 "이번 주 회독 캘린더 보기" 링크로 진입).
- 카드/시트 공통 radius: `radius-lg(16px)`, shadow: `shadow-card`.
- 빈 상태 일러스트: 마스코트 이미지 크기 80px, 불투명도 80%.
- 필터/탭 전환: 클라이언트 사이드 즉시 반영 (서버 round-trip 없이). 실제 데이터 연결은 P5~P6 단계.

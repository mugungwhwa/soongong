# 문제 리서치 에이전트 정의 v1.0

## 0. 한 문장

> 학생이 올린 수능 문제(사진 또는 텍스트)를 받아 **분해 → 단원 분류 → 변형 1개 생성 → 회독 스케줄화** 까지 한 번에 처리하는 에이전트.

---

## 1. 페르소나

- 한국 수능 출제 패턴에 능숙한 분석가
- 16-Agent 아키텍처의 입력 6 + 변형 8 = 14개 sub-agent의 **단일 dispatcher**
- MVP 1차에서는 V1(숫자 변형) + V2(요구값 변경)만 자신 있게 처리
- V3 이상 / 4과목 중 영어·국어·탐구 / 베리에이션 묶음 출제는 거절 (MVP 2차)

---

## 2. 언제 호출하는가 (trigger)

| 상황 | 호출? |
|---|---|
| 학생 앱에서 문제 사진 업로드 → POST /api/intake | ✅ 자동 호출 |
| 개발 중 수학 문제 텍스트 paste → "이 문제 분석해줘" | ✅ 수동 호출 |
| 변형 문제 1개 필요 → "이거 V1으로 바꿔줘" | ✅ 수동 호출 |
| 단원 분류 검증 → "이 문제 어느 단원이야?" | ✅ 수동 호출 |
| 베리에이션 묶음 5개 출제 | ❌ 거절 (MVP 2차) |
| 영어 지문 분석 | ❌ 거절 (MVP 2차) |

---

## 3. 도구 의존성

| 도구 | 용도 | MVP 1차 |
|---|---|---|
| **Claude Sonnet 4.6** (vision + JSON mode) | 사진 OCR + 분해 + 변형 생성 | ✅ |
| **Curriculum RAG** (Postgres FTS) | 단원/토픽 후보 lookup | ✅ (Task 5 게이트 통과 시) |
| **Mathpix OCR** | 수식 영역 정확도 폴백 | 옵션 (ENABLE_MATHPIX flag) |
| **Type Pattern RAG** | 유형별 변형 전략 lookup | ❌ (MVP 2차) |
| **Student Memory RAG** | 학생 오답 패턴 lookup | ❌ (MVP 2차, cold start) |

---

## 4. 폴백 룰

| 시그널 | 폴백 |
|---|---|
| OCR 정확도 < 90% (수학 점화식 10장 기준) | 학생에게 "사진이 흐려요" + 수동 텍스트 입력 폼 |
| Curriculum RAG Top-1 신뢰도 < 0.3 (모든 후보 < 0.3) | unit/topic을 "기타"로, routing_confidence ≤ 0.5 표시 |
| Curriculum RAG 미구축 (Task 5 게이트 미통과) | Claude의 일반 지식으로 단원 추정 + routing_confidence ≤ 0.6으로 명시 |
| 베리에이션 검증 필요(`validation_required=true`) | 결과를 admin 큐에 보냄 (MVP 2차에서 처리) |
| V3 이상 변형 요구 | 거절 + "MVP 2차에서 지원 예정" 메시지 |

---

## 5. 출력 검증 (반드시 통과)

에이전트 출력 JSON은 spec §4 출력 스키마와 **정확히** 일치해야 한다. 어긋나면:

1. **자동 retry 1회** — "출력을 §4 스키마에 맞춰 다시 작성" 프롬프트
2. **2회 실패 시 거절** — validation_required=true로 마킹, admin 큐로

스키마 검증은 Zod로 강제 (P3 plan Task 3 공통 유틸 사용).

## 6. 호출 경로 (production)

```
[학생 앱]
    │ 사진 업로드
    ▼
POST /api/intake → Supabase Storage
    │
    ▼
Edge Function: research-agent (이 에이전트)
    │
    ├─→ Curriculum RAG lookup (Postgres FTS) — Task 3
    ├─→ Claude Sonnet 4.6 vision call + JSON mode
    ├─→ Zod schema 검증
    │
    ▼
learning_objects + quests 테이블 insert
    │
    ▼
학생에게 분해 결과 카드 반환
```

## 7. 호출 경로 (개발/디버깅)

Claude Code 세션에서 invoke:

```
Agent(
  description: "수학 문제 1개 분석 및 V1 변형 생성",
  subagent_type: "문제-리서치-에이전트",
  prompt: "<spec §2 입력 JSON 또는 자연어로 문제 paste>"
)
```

로컬 subagent 파일 `~/.claude/agents/문제-리서치-에이전트.md`가 invoke 가능 형태.

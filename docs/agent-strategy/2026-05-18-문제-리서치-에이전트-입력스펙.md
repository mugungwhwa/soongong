# 문제 리서치 에이전트 — 입력 스펙 v1.0

## 0. 한 문장

> 학생이 올린 문제 사진을 받아서, **"이게 무슨 문제인지 분해"** 하고, **"숫자만 살짝 바꾼 변형 1개"** 를 만들어 회독퀘스트로 내보내는 에이전트.

---

## 1. 책임 범위 (MVP 1차 잠금)

### ✅ 한다
- **분해 (리서치)** — OCR로 문제 본문/선지/조건 추출 → 4과목 중 무엇인지 → 단원·유형·난이도(L1~L5) → 학생이 어디서 틀릴지 후보 2개까지
- **변형 (생성)** — V1(숫자만 바꿈) 또는 V2(묻는 항만 바꿈) 1개만
- **퀘스트화** — 1/3/7/14일 회독 스케줄 + 개념카드/OX/미니문항 중 1 format

### ❌ 안 한다 (MVP 2차로 미룸)
- V3 표현 변경 / V4 조건 역방향 / V5 4점보스형
- 제휴·기출 본문 직접 수입
- 베리에이션 묶음(같은 문제 5개 변형 한 번에 출제)
- Admin 큐레이션 검수 (P8 별도 트랙)

---

## 2. 입력 (학생/클라이언트 → 에이전트)

> "학생이 사진 한 장 + 과목 힌트 정도만 던진다"

```json
{
  "session_id": "uuid",
  "user_id": "uuid",
  "source_type": "photo | text",
  "subject_hint": "math | korean | english | inquiry | null",
  "image_url": "https://<supabase-storage>/...",
  "raw_text": null,
  "student_marks": {
    "highlights": [],
    "annotations": []
  }
}
```

**규칙:**
- `subject_hint`가 `null`이면 라우팅 에이전트가 confidence로 자동 판단
- `image_url`과 `raw_text` 중 최소 1개 필수
- `student_marks`는 미래용 슬롯 (MVP 1차에선 빈 배열 OK)

---

## 3. 프롬프트 골격 (Claude Sonnet 4.6 + JSON mode)

> "에이전트한테 던지는 시스템 프롬프트의 뼈대"

```
[SYSTEM]
너는 한국 수능 "문제 리서치 에이전트"다.
역할: 입력 문제 1개를 분해하고, V1 또는 V2 베리에이션 1개를 생성한다.

규칙:
1. subject_hint가 있으면 그 과목으로 우선, 없으면 라우팅 confidence ≥ 0.7로 판단
2. difficulty는 L1(쉬움)~L5(4점보스) 5단계
3. wrong_reason 후보는 최대 2개
4. 변형은 V1(숫자 swap) 또는 V2(요구값 변경)만 허용 — V3 이상 요청 시 거절
5. 출력은 반드시 §4 스키마 JSON

[USER]
<입력 JSON (§2)>

[ASSISTANT]
<출력 JSON (§4)>
```

---

## 4. 출력 (에이전트 → DB/클라이언트)

> "분해 결과 + 변형 1개 + 회독 스케줄을 한 묶음으로 돌려준다"

```json
{
  "analysis": {
    "subject": "math",
    "unit": "수열",
    "topic": "점화식",
    "question_type": "객관식",
    "difficulty_level": "L3",
    "wrong_reason_candidates": ["계산 실수", "조건 누락"],
    "routing_confidence": 0.92
  },
  "variation": {
    "variation_level": "V1",
    "variation_strategy": "numeric_swap",
    "stem": "...",
    "choices": ["...", "...", "...", "...", "..."],
    "answer_index": 2,
    "expected_correct_rate": 0.65,
    "validation_required": false
  },
  "quest": {
    "quest_type": "오늘의 회독",
    "format": "미니문항",
    "review_schedule_days": [1, 3, 7, 14],
    "xp_reward": 10,
    "memory_hp_reward": 5
  }
}
```

## 5. 게이트 (못 넘으면 차단 → 폴백)

| 게이트 | 기준 | 미달 시 처리 |
|---|---|---|
| **OCR 정확도** | 수학 점화식 10장 ≥ 90% | "사진이 흐려요" 안내 + 수동 입력 폼 |
| **라우팅 confidence** | ≥ 0.7 | "어떤 과목인가요?" 학생에게 물어봄 |
| **Compliance** | 저작권/개인정보 위반 0건 | 자동 거절 + 학생 안내 메시지 |
| **베리에이션 검증** | `validation_required=false` | true면 admin 큐로 보냄 (MVP 2차) |

---

## 6. 호출 경로 (실제 흐름)

```
   [학생 앱]
       │  사진 업로드
       ▼
   POST /api/intake
       │
       ▼
   Supabase Storage 저장
       │
       ▼
   Edge Function: research-agent ← 여기가 본 에이전트
       │
       ├─→ Claude Sonnet 4.6 (JSON mode)
       │
       ▼
   learning_objects 테이블 insert (analysis)
       │
       ▼
   quests 테이블 insert × 4건 (1일/3일/7일/14일)
       │
       ▼
   학생에게 "분해 결과 카드" 표시
```

---

## 7. 자주 묻는 경계 케이스

| 상황 | 처리 |
|---|---|
| 학생이 한 사진에 문제 3개를 올림 | OCR 단계에서 "문제 1개씩 분리해 주세요" 안내 (MVP 1차) |
| 손글이 풀이가 같이 찍힘 | `student_marks.annotations`에 OCR 결과만 보관, 분해엔 미사용 |
| 영어 지문이 너무 길어서 토큰 초과 | "긴 지문은 다음 버전 지원" 안내 + 1문장 단위로 자르도록 가이드 |
| V3 표현 변경을 요구함 | 거절 + "MVP 2차에서 지원 예정" 메시지 |
| 같은 문제 재업로드 | content_hash로 dedupe, 기존 learning_object 재사용 |

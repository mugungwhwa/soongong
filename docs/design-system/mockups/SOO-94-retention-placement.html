SOO-94 · 리텐션 제안 P1~P5 화면 배치 목업

SOO-93 경쟁 티어다운에서 도출한 리텐션 제안을, 현재 순공이 실화면 위에 어디에 어떻게 놓이는지 Before/After로 시각화. 색·로고·마스코트·토큰은 그대로 두고 배치·플로우·UX 패턴 레벨만 제안합니다.

Before = 현행 화면 After = 제안 배치(녹색 테두리) 1차 권장 고임팩트·저난이도(P1~P3) 2차 P4~P5
🔒 브랜드 락 준수: 모든 시안은 기존 토큰(#4CAF88 CTA / 크림 배경 / 듀공 "순공이") 안에서만 배치를 바꿉니다. 기억HP는 0–5 정수(백분율·하트 금지), 카피는 동반자 톤(공포·죄책감·passive-aggressive 금지), light-only(다크 네이비 0)·코어 회독 루프 무수정.
P1

비-처벌 빈 상태(Empty State)

1차 권장 화면: 홈(/today) 신규·due 없음 + 오답노트 빈상태

After — 제안 배치

안녕하세요, 민수님!

Zz

무엇이 바뀌나

  • 빈 화면 → 순공이(sleep mood)가 쉬는 비-처벌 카드로 교체
  • "밀렸다/놓쳤다" 대신 "쉬어가도 괜찮아요" 동반자 카피
  • 막다른 빈 상태 대신 가벼운 다음 행동 2개(새 문제 / 다시 보기)

듀오링고 매트릭스

  • 차용: 빈 상태에도 마스코트로 정서 유지
  • 거절: 빨강·죄책감·"연속 깨짐!" 압박 ❌ 반영 안 함
동반자 톤 ✅ 마스코트 mood 일치(휴식=sleep) ✅ 홈 게임성 강도 ≤50% 캡 내 하트·백분율 0 ✅

요약 — P1~P3(1차 권장, 고임팩트·저난이도)은 홈(/today) 한 화면에서 빈상태·리마인드·due 정수로 묶이고, P4~P5(2차)는 보호권·숙련도 시각화로 스트릭/여정 화면에 붙습니다. 전부 기존 토큰·마스코트 안의 배치·카피·UX 패턴 변경이며 코어 회독 루프·문제생성 로직은 손대지 않습니다.

SSoT 메모 — 이 파일의 색 값은 시안 표시용 미러일 뿐, 실제 구현은 apps/web/src/shared/styles/tokens.css 변수를 참조합니다. 본 목업은 "어디에 어떻게 놓일지" 합의용이며, 구현 착수는 별도 티켓.

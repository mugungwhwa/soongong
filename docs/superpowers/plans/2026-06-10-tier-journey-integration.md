# 티어 여정 통합 기획 v0.1 (2026-06-10)

> 기존 유저 진입 = "지금 어디(현재 등급) → 어디로(다음 등급) → 눌러서 오늘 학습". 8 Ball Pool식 단계-카드 진행감을 차용하되 베팅·도박 메커닉은 배제(멤버십/여정 톤). design-system-lock v1.0(Ocean/로고A) 위에서 구현.

## 1. 목표 (한 줄)
기존 유저가 앱을 열면 현재 등급·다음 목표가 한눈에 보이고, 거기서 오늘의 회독으로 자연스럽게 진입.

## 2. 진입 분기
```
로그인 후
 ├─ 신규 유저  → /onboarding (기존 그대로)
 └─ 기존 유저  → /today (여정 hero + 오늘의 회독 2단)
```
- 로비 강제 X (듀오링고 저마찰). 새 라우트 안 만들고 `/today`를 "여정 hero(상단) + 기존 오늘의 회독(하단)" 으로 확장 → 라우팅·충돌 리스크 최소.
- 전체 6장 여정은 "내 여정" 한 탭 더(후속).

## 3. 화면
- 상단 hero: 현재 등급 카드 "지금 여기" + 다음 등급까지 진행바(회독/XP 남음) + CTA "오늘 회독 시작하기".
- 하단: 기존 오늘의 회독(GameBar·마스코트·퀘스트 3개) 유지.

## 4. 잠긴 SSoT 정합
- 등급 6단(순공입문→전설, §6-1) / 게임화 v1.0 = 데이터 그대로, 표현만 추가.
- design-system-lock 오션·로고A = 이미 main 반영. 토큰(var(--color-*))만 사용, 하드코딩 hex 금지(lint:tokens).

1. **[이번]** `widgets/tier-journey-hero` 컴포넌트 — 등급/XP 바인딩, hero UI. `views/today` 상단에 삽입. 분기 로직 없음 → 충돌 0.
2. 기존/신규 분기 적용 (§6-1 결정 후).
3. "내 여정" 전체 6장 화면(탭).
4. lint·tsc·빌드 + 반응형 확인.

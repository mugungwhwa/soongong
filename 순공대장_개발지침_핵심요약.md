# 순공대장 개발 지침 핵심요약

## 1. 제품 정의

순공대장은 수능생이 인강을 보고 끝내지 않도록, AI가 문제사진·오답·인강기록·캡처·메모를 매일 복습 퀘스트로 바꿔주는 듀오링고형 AI **회독 리텐션 엔진**이다.

핵심은 문제를 대신 풀어주는 AI가 아니라, 학생이 다시 풀게 만드는 AI다.

> **콴다는 학생의 '막힘'을 풀고, 순공대장은 학생의 '까먹음'을 푼다.**

기능 비교가 아니라 **무엇을 학습이라고 믿는가**의 차이. 학술적 근거는 Karpicke & Roediger (2008, *Science*): 단순 학습 그룹은 1주 뒤 80%를 까먹었고 인출 연습 그룹은 80%를 기억했다 — 같은 시간, 4배 차이. 상위 사업 SSoT는 `2026-05-19-순공대장_전략_정리.md` 참조.

## 4. AI Agent 원칙

Agent는 다음 구조를 따른다.

```text
Source Intake
→ Compliance Gate
→ OCR / Parsing
→ Subject Routing
→ Question Analysis
→ Question DNA
→ Curriculum RAG / Type Pattern RAG / Student Memory RAG
→ Difficulty Agent
→ Variation Agent
→ Solver / Evaluation
→ Review Scheduling
→ Quest / Game Agent
```

## 5. RAG 원칙

기출 원문 RAG가 아니라 수능 유형 카드 RAG로 간다.

- Curriculum RAG: 교육과정, 단원, 개념 범위
- Type Pattern RAG: 수능 유형 카드, 문항 DNA, 변형 축, 오답 패턴
- Student Memory RAG: 학생별 오답, 풀이반응, 숙련도, 회독 이력

**Student Memory RAG는 복리 자산**: 1주 차 단순 태그 → 1개월 패턴/빈도 → 3개월 풀이 사고 모델 → 6개월 유사 사고 학생 데이터로 선제 추천 가능. 개별 학생을 깊게 알수록 회독 정확 → 머무름 → 데이터 축적되는 ML flywheel. 콴다/Anki가 못 만드는 모트.

**Onboarding 베이지안 prior**: `P(학생 실력 | 진단평가) ∝ P(진단평가 | 학생 실력) × P(학생 실력)`. Prior가 정확할수록 진단 문항 수 감소 — 약한 prior 50문제 → 90% 이탈, 강한 prior 10문제 → onboarding 통과. 계층적 베이지안 모델 + CUPED (학생별 사전 실력으로 variance 감소) 적용. 상세는 전략정리 §3.4-3.7.

## 6. 오답회수 모드 원칙

틀린 직후 해설만 보여주고 끝내지 않는다.

```text
틀림
→ 오답 원인 진단
→ 짧은 해설
→ 비슷한 문제로 바로 회수하기
→ 성공 시 XP/숙련도/망각위험 감소
→ 실패 시 L1 개념 회수로 하향
```

CTA는 “유사문제 풀기”보다 “비슷한 문제로 회수하기”, “오답 바로 잡기”처럼 감정이 있는 문구를 쓴다.

## 7. 난이도 원칙

문항 난이도와 학생 체감 난이도를 분리한다. MVP는 L1~L5 5단계로 시작한다.

- L1: 개념 회수, OX, 빈칸
- L2: 단순 적용
- L3: 수능 3점형
- L4: 준 4점형
- L5: 4점보스

오늘의 회독은 L1~L3 중심, 보스전은 선택형으로 둔다.

## 8. 기술 스택 기본값

- Web: Next.js, TypeScript, App Router, FSD 2.1, Tailwind, shadcn/ui
- App: Expo React Native, Expo Router, TypeScript, NativeWind/Unistyles
- Backend: Supabase, PostgreSQL, Storage, RLS, Edge Functions, pgvector, Cron
- AI: LLM API + Agent Orchestrator + RAG Retrieval + Quality Evaluation
- Pad solving: Web MVP는 tldraw/Excalidraw/Konva 후보, App은 React Native Skia, 장기적으로 PencilKit/MyScript/Mathpix 검토

## 9. 보안/법무 원칙

- 원본 문제사진/인강캡처는 최소 저장하고, 파생 학습 데이터 중심으로 누적한다.
- 유료 인강 전체 녹화, 자동 캡처, 영상 저장은 금지 방향.
- 기출 원문 대량 저장/노출은 피하고, 유형 카드화 후 원문 비노출 구조로 간다.
- 학생 데이터는 RLS 기반으로 본인 데이터만 접근하게 한다.
- AI 분석 결과는 오류 가능성을 고지하고, 사용자 수정/오류 신고 기능을 제공한다.

## 11. 데이터 아키텍처 (전략정리 §3.9)

**하이브리드 모델 — Postgres truth + pgvector 검색**:
- **Postgres** = 진실의 원본 (온톨로지 트리, 학생 메모리, 회독 스케줄 — 사람이 검수 가능한 정형 구조)
- **pgvector** = 검색용 (학습 객체 임베딩, 의미 기반 유사도 — 변형/유사문항 검색)

→ P3 sub-plan에 명시 박혀있어야 함. 둘은 분리 운영, 검색은 pgvector 결과를 Postgres truth로 join.

## 12. 온톨로지 구조 (전략정리 §3.3, §3.8)

```
과목 → 영역 → 단원 → 개념 → 유형 → 풀이전략 → 오답패턴
                                              ↓
                                        변형축 (조건/수치/제시방식)
```

이게 정확하지 않으면 "비슷한 문제"가 랜덤 문제가 됨 → 한 번 풀어보고 "도움 안 되네" → 이탈.

**회색지대 처리**: 한 문제가 여러 단원에 걸칠 때 — 초기엔 **단일 분류 + 사용자 수동 다중 태그**. AI는 단일 추천만, 1년 데이터 후 다중 분류 자동화.

## 14. 버티컬·글로벌 확장 (전략정리 §4, §6)

**버티컬 확장 순서** (Mikhoon 도메인 지식 누적 방향):
- 1년 차: **수능 집중 + 검증** (본인 학원 베타)
- 2년 차: **공시 확장** (같은 엔진, 새 온톨로지)
- 3년 차+: 자격증 → 어학 → 글로벌

각 버티컬마다 OCR/태깅 fine-tuning + 온톨로지 새로 만들어야 함 — 이게 진짜 모트(1~2년 걸림).

**글로벌 확장 순서**:
- 1순위 일본 (3-5년차): 사교육 9조엔, 입시 문화 90% 동일, 한자권
- 2순위 동남아 (5년차): SDK/라이센스 B2B2C
- 3순위 미국 Pro Exam (5-7년차): MCAT/Bar/CPA/CFA만 ($50-200/월)
- 보류: K-12 미국(rote learning 거부), 중국(雙減政策)

**첫 18개월 한국 집중 — 너무 빨리 글로벌 시도하면 양쪽 다 망함** (Riid, 콴다 글로벌 실패 사례).

**브랜드**: 글로벌 그대로 "Soongong" / 영문 부제 "The Recall Engine for Test Prep".

# 순공대장 개발 지침 핵심요약

## 1. 제품 정의

순공대장은 수능생이 인강을 보고 끝내지 않도록, AI가 문제사진·오답·인강기록·캡처·메모를 매일 복습 퀘스트로 바꿔주는 듀오링고형 AI 수능 회독 앱이다.

핵심은 문제를 대신 풀어주는 AI가 아니라, 학생이 다시 풀게 만드는 AI다.

## 2. 브랜드/UX 기준

- 브랜드명: 순공대장
- 마스코트: **순공이** (가안 — 듀공 형태 일러스트, 이름·외형 모두 가안이며 추후 확정. 시안은 `app_UI.png` / `web_ui.png` 참조)
- UI 명칭: "회독" = "회독퀘스트" (사용자에게 노출되는 라벨은 **"회독퀘스트"** 권장, 백서·기술 문서에서는 "회독" 그대로 사용 OK)
- 핵심 UX: 오늘의 회독길
- 디자인 톤: 밝은 크림/민트 배경, 둥근 카드, 듀오링고형 세로 경로, 과하지 않은 게임성
- 홈의 목적: 대시보드가 아니라 오늘 해야 할 회독 3개를 바로 시작하게 만드는 것

## 3. 핵심 기능

1. 문제사진 업로드
2. 인강명·단원명·캡처·메모 입력
3. AI 과목/단원/유형/오답원인 태깅
4. 1일/3일/7일/14일 회독 예약
5. 오늘의 회독 생성
6. 오답회수 모드
7. 망각방어전
8. XP, 기억 HP, 스트릭, 등급, 순공리그
9. 패드 풀이과정 캔버스 및 풀이로그 저장

## 4. AI Agent 원칙

Agent는 다음 구조를 따른다.

```text
Source Intake
→ Compliance Gate
→ OCR / Parsing
→ Subject Routing
→ Question Analysis
→ Question DNA
→ Curriculum RAG / Type Pattern RAG / Student Memory RAG
→ Difficulty Agent
→ Variation Agent
→ Solver / Evaluation
→ Review Scheduling
→ Quest / Game Agent
```

## 5. RAG 원칙

기출 원문 RAG가 아니라 수능 유형 카드 RAG로 간다.

- Curriculum RAG: 교육과정, 단원, 개념 범위
- Type Pattern RAG: 수능 유형 카드, 문항 DNA, 변형 축, 오답 패턴
- Student Memory RAG: 학생별 오답, 풀이반응, 숙련도, 회독 이력

## 6. 오답회수 모드 원칙

틀린 직후 해설만 보여주고 끝내지 않는다.

```text
틀림
→ 오답 원인 진단
→ 짧은 해설
→ 비슷한 문제로 바로 회수하기
→ 성공 시 XP/숙련도/망각위험 감소
→ 실패 시 L1 개념 회수로 하향
```

CTA는 “유사문제 풀기”보다 “비슷한 문제로 회수하기”, “오답 바로 잡기”처럼 감정이 있는 문구를 쓴다.

## 7. 난이도 원칙

문항 난이도와 학생 체감 난이도를 분리한다. MVP는 L1~L5 5단계로 시작한다.

- L1: 개념 회수, OX, 빈칸
- L2: 단순 적용
- L3: 수능 3점형
- L4: 준 4점형
- L5: 4점보스

오늘의 회독은 L1~L3 중심, 보스전은 선택형으로 둔다.

## 8. 기술 스택 기본값

- Web: Next.js, TypeScript, App Router, FSD 2.1, Tailwind, shadcn/ui
- App: Expo React Native, Expo Router, TypeScript, NativeWind/Unistyles
- Backend: Supabase, PostgreSQL, Storage, RLS, Edge Functions, pgvector, Cron
- AI: LLM API + Agent Orchestrator + RAG Retrieval + Quality Evaluation
- Pad solving: Web MVP는 tldraw/Excalidraw/Konva 후보, App은 React Native Skia, 장기적으로 PencilKit/MyScript/Mathpix 검토

## 9. 보안/법무 원칙

- 원본 문제사진/인강캡처는 최소 저장하고, 파생 학습 데이터 중심으로 누적한다.
- 유료 인강 전체 녹화, 자동 캡처, 영상 저장은 금지 방향.
- 기출 원문 대량 저장/노출은 피하고, 유형 카드화 후 원문 비노출 구조로 간다.
- 학생 데이터는 RLS 기반으로 본인 데이터만 접근하게 한다.
- AI 분석 결과는 오류 가능성을 고지하고, 사용자 수정/오류 신고 기능을 제공한다.

8. XP / 스트릭 / 기억 HP
